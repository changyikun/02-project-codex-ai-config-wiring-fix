## 0. 现状与边界

### 已存在且已落库的关键配置（摘要）
- 时间推进：7 格=1 旬；3 旬=1 月；12 月=1 年；时间段统一为：清晨/上午/中午/下午/傍晚/夜晚/深夜。
- 属性与换算：主属性点→100，福德→10，技能等级→10；并提供无负数的整数化输出函数。
- 路线与 NPC：4 条路线配置、11 个具名 NPC 初始表。
- 地点开放：已建立 `LOCATION_OPEN_TIME`（含“冷宫/养心殿”新增点）与宴会拒绝惩罚 `EVENT_PENALTY.banquetRefuse(20~30)`。

### 本文“独立补全”的定义
- 文档中明确给出数值/范围的：直接固化。
- 文档中未给出精确数值但已给出机制的：补齐为“可配置参数”，并给出推荐默认值与理由。
- 文档中未定义机制但会影响循环闭环（经济/成长/难度/负反馈）的：以“最小可玩闭环”为目标补齐一套默认参数。

## 1. 补全内容清单（可直接落成配置）

### 1.1 经济循环（新增：月度俸禄、银两收支口径）
目的：避免玩家银两“只出不进”导致玩法塌陷；同时防止银两“滚雪球”破坏难度曲线。

**新增规则：月结算银两俸禄（按位分）**
- 俸禄发放时机：每月结算时（3 旬结束后）。
- 俸禄范围表（单位：两/月）
  - 皇后：`[220, 280]`
  - 皇贵妃（特殊）：`[180, 240]`
  - 贵妃：`[160, 210]`
  - 德妃/淑妃/贤妃：`[120, 170]`
  - 妃：`[90, 130]`
  - 九嫔：`[70, 110]`
  - 贵嫔：`[55, 90]`
  - 婕好：`[40, 70]`
  - 容华：`[35, 60]`
  - 嫔：`[28, 50]`
  - 贵人：`[20, 40]`
  - 美人：`[15, 30]`
  - 才人：`[10, 22]`
  - 常在：`[8, 18]`
  - 御女：`[6, 14]`
  - 选侍：`[4, 10]`
  - 答应：`[2, 8]`
  - 更衣：`[1, 6]`
  - 官女子：`[0, 3]`

**理由（耦合点与策略影响）**
- 与既有经济支出耦合：食物（2~3 两）、御书房贿赂（10 两）、商贩礼物（70/130/180/250 两）、毒药（150/250/500 两）、宴会（10/20/40 每人）。
- 俸禄表设计为“可攒但不富裕”：低位分仍可通过 1-2 月积累完成一次关键支出（如 10 两贿赂/小额礼物），高位分可更频繁参与宴会/送礼但仍难以无脑购买红礼与鹤顶红。
- 对玩法循环的影响：鼓励“月度规划”（本月是否存钱冲宴会/送礼/贿赂），并为低位分路线提供可恢复的经济底盘，降低早期卡死。

### 1.2 地点事件参数（补齐：御书房/寺庙/妙音堂等收益与概率）

#### 御书房（研磨/诉苦/美言/撒娇）
**新增参数**
- “研磨”政治成长：`POLITICS_GAIN_RANGE = [1, 3]`（点/次）
- “诉苦”目标宠爱降低：`TARGET_FAVOR_DELTA_RANGE = [-5, -2]`
- “美言”目标宠爱提升：`TARGET_FAVOR_DELTA_RANGE = [2, 5]`
- “撒娇”自身宠爱提升：`SELF_FAVOR_DELTA_RANGE = [2, 5]`
- 诉苦失败条件（文档口径固化为可计算条件）：当目标宠爱 > 玩家宠爱 且目标处于“盛宠及以上”，失败概率上升（见 1.4 宫斗/互动通用判定）。

**理由**
- 研磨与政治“后期内容”耦合：政治越高越多剧情分支（路线结局门槛），因此每次 1~3 点可支撑中后期在 6~10 个月内达成 80~95 的高门槛，但不会在 1~2 个月内爆表。
- 诉苦/美言/撒娇直接作用“宠爱→侍寝/封号/策略资源”，区间给到 2~5 以保证动作有感，同时限制单次波动避免滚雪球。

#### 妙音堂（听曲）
**新增参数**
- 压力降低：已固化为 `STRESS_REDUCE_LISTEN_MUSIC = 4`（点/次）。
- 新增“听曲冷却”：`MUSIC_HALL_COOLDOWN_SLOTS = 2`（同一旬内最多触发 1 次；用 2 格冷却实现）

**理由**
- 压力是“负面反馈系统”，若可无限刷会破坏约束；冷却保证“偶尔救急”，不成为主策略。

#### 宝华殿（礼佛）
**新增参数**
- 福德提升：`FORTUNE_GAIN_RANGE = [1, 3]`（游戏值/次）
- 福德<0 的强惩罚（已有机制）：受孕强制失败；若已怀孕次月必流产并清零怀孕状态。

**理由**
- 福德直接等价为“受孕概率值”，所以提升必须缓慢；每次 1~3 点需要持续投入时间成本，形成“要孩子/要权力/要经济”的权衡。

#### 寺庙外出（消耗）
**新增参数**
- 体力消耗与时间：已固化为体力 3、时间 1。
- 新增“外出随机事件权重”：`TEMPLE_EVENT_WEIGHTS = { lucky: 0.15, neutral: 0.7, bad: 0.15 }`

**理由**
- 外出作为“高体力消耗的波动收益点”，用于打破数值单调；坏事件防止稳赚。

### 1.3 宴会参与/拒绝（补齐：文档缺口的“提醒/拒绝类型差异”）
已固化：拒绝参加宴会声望扣除范围 `[20, 30]`。

**新增参数**
- 宴会提醒时段：`BANQUET_REMINDER_SLOT = '下午'`
- “告病拒绝”惩罚较轻：`BANQUET_SICK_REFUSE_PENALTY_RANGE = [10, 18]`
- “无故不去”惩罚按既定：`BANQUET_NO_SHOW_PENALTY_RANGE = [20, 30]`

**理由**
- 给玩家一个“资源不足/状态不佳”的软退出（告病），降低被强迫参与的负面体验，同时保留成本（仍掉声望）让选择有重量。

### 1.4 宫斗/互动判定（新增：统一成功率公式与防滚雪球）
目的：将“口角/陷害/下毒/侍寝互动”从纯随机变为可解释、可调参、可测试的统一判定框架。

**统一判定输出**
- `successProbability ∈ [0.05, 0.95]`（硬夹逼，避免 0% 与 100% 造成体验崩坏）

**通用成功率公式（建议默认）**
- `p = clamp(0.05, 0.95, sigmoid( base + wIntrigue*Δintrigue + wRank*Δrank + wFavor*Δfavor + wRandom*ε ))`
  - `sigmoid(x)=1/(1+e^-x)`
  - `ε ~ Uniform(-1, 1)`

**推荐权重（默认值）**
- `base = 0`
- `wIntrigue = 0.004`（心计每差 250 → 影响约 1 个单位输入）
- `wRank = 0.25`（位分每差 1 级 → 约 0.25 输入）
- `wFavor = 0.015`（宠爱每差 20 → 约 0.3 输入）
- `wRandom = 0.45`

**防滚雪球（新增硬规则）**
- “连续成功衰减”：同类型交互连续成功时，下一次 `base -= 0.35`，最多叠 2 层；失败清零。
- “月度惩罚上限”：同一月因宫斗导致的声望下降不超过 `300`（超过部分折算为压力增加）。

**理由**
- 与现有体系耦合：心计/位分/宠爱是核心资源，权重让“练属性/升位分/争宠”都能影响胜率；随机项保证戏剧性。
- 策略影响：玩家可以通过提升心计与位分稳定胜率，但无法完全规避风险；连续成功衰减抑制滚雪球。

### 1.5 成长节奏（补齐：技能/主属性提升的行动收益）
文档已有：技能 0~50 每次 2~4 点；>50 每次 1~2 点。需要补齐“每类行动对应的技能成长映射”。

**新增映射（默认）**
- 看书/读书 → 诗词（或才学）`+2~4 / +1~2`
- 乐理/听曲 → 乐理 `+2~4 / +1~2`
- 字画 → 丹青 `+2~4 / +1~2`
- 刺绣/女红刺绣 → 刺绣 `+2~4 / +1~2`
- 调香/会诊/太医院学习 → 药理 `+2~4 / +1~2`（且药理≤5 为全局硬上限，路线例外照旧）
- 御书房研磨 → 政治 `+1~3`（政治为后期；进入门槛由地点机制控制）

**理由**
- 与玩法循环耦合：行动→成长→触发剧情→状态变化/获得物品；映射明确后，AI 文案池可按“行为类型+阶段”生成细腻文本。

### 1.6 剧情分支补齐（新增：可配置触发口径与缺失节点）
目的：让路线结局与关键事件具备统一的“节点/旗标”表述，便于热更新与回滚。

**新增统一旗标**
- `flag.event.nightly-service`（侍寝触发）
- `flag.event.banquet.palace`（宫宴触发）
- `flag.event.banquet.family`（家宴触发）
- `flag.event.selection`（选秀触发）
- `flag.item.tiger-tally`（虎符）
- `flag.item.evidence`（证物）
- `flag.quest.fake-death-medicine`（假死药完成）

**理由**
- 将“事件条件表达式”从散落文本变为统一键，减少剧情分支的维护成本，并支持热更新替换为真实剧情节点系统。

## 2. 可玩性保障清单

### 2.1 平衡性测试用例（建议最小集）
- 经济闭环
  - 低位分开局（银两≤100）在 2 个月内能完成至少 1 次关键支出：10 两贿赂或 70 两绿礼物（二选一）。
  - 高位分开局（月俸≥160）连续 2 个月“送红礼+鹤顶红”会出现明显机会成本（体力/声望/压力）而非无代价通关。
- 成长节奏
  - 技能从 0→50：平均 15~25 次行动可达（符合 2~4 的收益预期）。
  - 技能从 50→80：平均 20~30 次行动可达（符合 1~2 的收益预期）。
  - 政治从 0→80：至少需要 30 次以上“御书房研磨”且受进入概率/门槛约束，避免早期直达结局门槛。
- 风险系统
  - 压力>80 的持续月惩罚能在 2~4 个月内显著影响寿命（强约束），但提供“可逆手段”（听曲/温泉/降低宫斗频率）避免必死。
- 宫斗判定
  - Δintrigue=+400、Δrank=+2、Δfavor=+30：成功率应显著高于 50%（建议 70% 左右）。
  - Δintrigue=-400、Δrank=-2、Δfavor=-30：成功率应显著低于 50%（建议 30% 左右），但不为 0%。

### 2.2 极端情况边界值（必须钳制）
- 声望：`[-2000, 5000]`（达到边界后溢出处理：截断，不继续增长/下降）
- 宠爱/好感/野心/福德：按各自域截断
- 成功率：`[0.05, 0.95]`
- 银两：不得为负；支出不足则动作不可执行或触发“赊欠/惩罚”事件（需明确口径）

### 2.3 玩家成长节奏验证表（示例）
- “路线三 影落掖庭”目标：10 年内达妃位+翻案
  - 关键里程碑：第 12 月达到声望≥750（容华），第 24 月达到≥1100（贵嫔），第 36~48 月达到≥1500（妃）
  - 若达不到：检查俸禄、宴会收益、宫斗成功率权重与压力惩罚是否过强

### 2.4 负面体验预警指标（上线后监控）
- 早期卡死：第 1~3 月“银两<10 且体力<3 且压力>60”的玩家占比 > 8%
- 雪球：第 3 月内“声望>1800 且银两>1000 且压力<20”的玩家占比 > 3%
- 逆反馈过强：连续 2 月压力>80 的玩家中，平均寿命损失导致的提前失败率 > 15%
- 事件惩罚过痛：宴会拒绝导致声望跌破 0 的比例 > 10%

## 3. 版本回溯与 24 小时回滚方案（无损）

### 3.1 配置版本化策略
- 将“可热更新参数”抽离为独立 JSON（例如 `server/config/game/balance.json`），并记录：
  - `version`
  - `generatedAt`
  - `checksum`
- 前端运行时从服务端拉取（失败则回退到内置 TS 默认值）。

### 3.2 回滚流程（24 小时内）
1) 保留上一个稳定版本配置文件（`balance.v_prev.json`）。
2) 热更新发布新配置（`balance.v_new.json`）。
3) 监控预警指标触发阈值后：
   - 立即切换到 `balance.v_prev.json`（不改存档结构，仅改运行时读取的参数）
   - 记录 `rollbackReason` 与 `rollbackAt`
4) 对“已受影响的存档数据”采取无损处理：
   - 不回写玩家数值（避免存档被重算污染）
   - 仅修正未来事件的计算参数（避免二次伤害）

### 3.3 热修策略（回滚后再迭代）
- 若问题来自单一系统（例如宴会惩罚过痛）：
  - 仅调整该系统 JSON 子树（如 `eventPenalty.banquetRefuse`），并保持其余系统参数不动
- 若问题来自耦合（例如俸禄+宴会收益导致雪球）：
  - 同步下调“俸禄上限”或提高“压力增长/冷却成本”，以保持曲线连续
