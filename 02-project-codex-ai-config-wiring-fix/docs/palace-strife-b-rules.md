# 宫斗系统 B 版硬规则定稿与测试文档

## 一、文档定位

这份文档用于单独定义宫斗系统的硬规则，不依赖 AI 判定。AI 在这一模块中只负责后续剧情播报、娇娇通报、皇帝与妃嫔对白，以及纪事文案补充，不参与任何真实检定结果。

本版本的目标是三点：第一，能够快速实现；第二，逻辑链条足够清晰，没有明显硬伤；第三，后续方便调参，不会因为一个公式改动就把整套系统推翻。

这份文档采用的是我建议你优先落地的 **B 版：对抗平衡版**。它和你最初设想相比，最大的优化是：取消“简单 / 一般 / 困难”作为前置的硬门槛判定，只保留它们作为最终展示标签。这样既保留了玩家对事件难易程度的直观感受，又避免了原本那种从 96% 突然掉到 16% 的断崖式成功率。

## 二、B 版系统核心原则

### 1. 宫斗行为分为两类

玩家当前可执行的宫斗行为只有两类：

- 造谣
- 下毒

其中，造谣内容固定为：

- 欺凌宫人，记为轻事件
- 奢侈浪费，记为轻事件
- 与人偷情，记为中事件
- 意图谋逆，记为重事件
- 不敬先祖，记为重事件

下毒内容固定为：

- 陨颜丹，记为轻事件
- 麝香，记为中事件
- 鹤顶红，记为重事件

### 2. 结算时间固定

玩家在白天选择造谣或下毒后，该行为不会立刻出结果，而是统一在 **当旬夜晚** 进行系统判定。到 **下一旬开始时**，由宫女娇娇来通报宫中消息。这样可以让宫斗结果和日常行动错开，形成明确的“做局”和“听回报”的节奏。

### 3. 每次宫斗固定进行两次检定

每次造谣或下毒，系统都会进行两次独立检定。为了避免和原方案一样变成两次重复抽奖，这里明确将两次检定分工如下：

- 第一次检定：**行动检定**
  - 负责判断这次造谣或下毒是否真正生效。
- 第二次检定：**隐匿检定**
  - 负责判断这次行为是否暴露，是否进入调查。

由此，系统会稳定出现四种结果：

- 行动成功，隐匿成功：事件生效，且未暴露
- 行动成功，隐匿失败：事件生效，但进入调查
- 行动失败，隐匿成功：事件未生效，仅损失一次性道具或行动机会
- 行动失败，隐匿失败：事件未生效，但仍然进入调查

这样做的好处是，玩家能够很清楚地理解失败到底发生在哪一环，而不是像旧规则那样出现三层检定互相覆盖的问题。

## 三、B 版正式公式

## 3.1 基础属性口径

### 造谣攻击值

造谣完全由心计主导。

`造谣攻击值 A = 玩家心计 / 10`

因为心计范围是 `0~1000`，所以造谣攻击值范围自然落在 `0~100`。

### 下毒攻击值

下毒完全由药理主导。

`下毒攻击值 A = 玩家药理`

药理本身就是 `0~100`，因此无需二次换算。

### 目标防御值

造谣和下毒的目标防御值并不相同。

造谣时：

`目标防御值 D = 目标心计 / 10 + 目标宠爱 / 20`

这里的逻辑是：目标越心思缜密，越难被人编排成功；目标越受宠，谣言也越难直接站住脚。

下毒时：

`目标防御值 D = 目标心计 / 10 + 目标宠爱 / 20 + 目标药理 / 4`

这里多出的药理项，代表目标本人越懂医理、越懂药物，越不容易被轻易得手。

## 3.2 合谋规则

若玩家与某位妃嫔好感度大于 `60`，则视为可交好，可以选择合谋。

合谋时不直接让成功率翻倍，而是改成一个温和但有效的公式：

`合谋后攻击值 = floor((玩家攻击值 + 合谋者对应攻击值) / 2) + 10`

额外效果：

- 隐匿检定 `+5`
- 若后续案件成立，初始获罪率 `+10`

这里的设计思路是：合谋应当确实提高成功率，否则玩家不会想用；但合谋不能只是白送加成，所以一旦被查，案件风险也会更高。

## 3.3 嫁祸规则

嫁祸仍然保留，但不再通过“上调一级难度”实现，因为 B 版不采用前置档位判定。嫁祸的影响直接写进公式里。

嫁祸时：

- 行动检定额外 `-12`
- 隐匿检定额外 `-15`
- 若案件成立，初始获罪率 `+15`
- 每旬调查增长额外 `+5`

这表示：嫁祸不是不能做，但它一定会更难、更危险，且一旦暴露，反噬会更明显。

## 3.4 行动检定公式

`行动成功率 = 50 + A - D - 事件修正 - 嫁祸修正`

其中：

- 轻事件修正：`0`
- 中事件修正：`5`
- 重事件修正：`10`
- 嫁祸修正：若嫁祸则 `12`，否则为 `0`

最终成功率固定限制在 `10~90` 之间。

这个公式的意义非常直接：玩家攻击值越高、目标防御值越低，就越容易成功；事件越重、嫁祸越多，成功率就越低。

## 3.5 隐匿检定公式

`隐匿成功率 = 50 + 玩家心计 / 12 - 目标心计 / 15 + 玩家宠爱 / 8 + 合谋修正 - 事件修正 - 嫁祸修正`

其中：

- 轻事件修正：`5`
- 中事件修正：`10`
- 重事件修正：`15`
- 合谋修正：若合谋则 `+5`，否则为 `0`
- 嫁祸修正：若嫁祸则 `15`，否则为 `0`

最终隐匿成功率固定限制在 `5~85` 之间。

这个公式体现的逻辑是：隐匿主要看心计，但玩家宠爱越高，也越有机会在事情爆发后被偏袒、被压下去。反过来，目标越心计高，越容易察觉端倪。重事件和嫁祸，则都会显著拉低隐匿成功率。

## 3.6 展示难度标签

为了保留“简单 / 一般 / 困难”的直观反馈，系统仍然会给玩家一个难度标签，但它只是展示层，不再反向改变检定公式。

根据最终行动成功率显示：

- `70~90`：简单
- `40~69`：一般
- `10~39`：困难

这样玩家仍然能看到系统评价，但内部公式是连续变化的，不再出现原版那种断崖式概率。

## 四、调查、嫌疑人与定案率规则

## 4.1 什么时候进入调查

只要 **隐匿检定失败**，无论行动是否真正生效，系统都会进入调查阶段。调查最长持续三旬，期间“宫斗事务”的调查页显示该案件、嫌疑人和当前定案率。

## 4.2 初始定案率

进入调查后，系统先按以下公式计算实际发起者的初始定案率：

`实际发起者初始定案率 = 基础值 + (100 - 隐匿成功率) / 2 + max(0, 目标宠爱 - 20) / 2 + 嫁祸值 + 合谋值 - 发起者宠爱 / 5`

其中：

- 轻事件基础值：`20`
- 中事件基础值：`40`
- 重事件基础值：`65`
- 嫁祸值：若嫁祸则 `15`
- 合谋值：若合谋则 `10`

最终初始定案率限制在 `20~95` 之间。

v0.5.1 起，案件不再只有一个全局获罪率。每个进入调查的案件生成最多三名嫌疑人：

- 实际发起者优先进入
- 被嫁祸者优先进入，且初始定案率至少 `70`
- 玩家如果是被害者或被嫁祸者，必须进入
- 其余名额从存活非冷宫妃嫔中按动机补足
- `convictionRate` 只同步最高嫌疑人的定案率，作为展示字段

这个公式的设计重点在于：

- 隐匿失败得越彻底，实际发起者初始定案率越高
- 目标越受宠，案件越难压
- 发起者越受宠，越有机会得到包庇
- 合谋和嫁祸都不是纯加成，它们会在暴露后反向提高案件风险

## 4.3 每旬增长

调查期间，每过一旬，所有嫌疑人的定案率都会自动增长。

- 轻事件：每旬 `+8`
- 中事件：每旬 `+14`
- 重事件：每旬 `+20`
- 被嫁祸者：额外每旬 `+5`

任一嫌疑人的定案率达到 `100` 时，案件进入 `pending_verdict` 待裁断，不立即定罪，也不立即扣声望 / 宠爱 / 压力。三旬结束后仍无人达到 `100`，则案件视为疑案，不了了之。

## 4.4 调查页干预

调查期间，玩家可以在“宫斗事务”的调查页花银两干预任意嫌疑人。

- `20 银两 = 指定嫌疑人定案率 -5`
- `20 银两 = 指定嫌疑人定案率 +5`
- 推高到 `100` 时只进入待裁断，处罚必须等养心殿或内廷裁断完成后才落地

这个换算关系简单、稳定，方便前期直接实现，也方便玩家理解“我要花多少钱才能把某个人压下去或推上去”。

## 五、被诬陷时的自证规则

如果玩家被他人诬陷，则进入一轮单独的自证检定。这里保留你想强调的“宠爱 + 容貌 + 气质更容易让人心软”的逻辑，但不再只看宠爱一个值。

正式公式为：

`自证成功率 = 25 + 玩家宠爱 / 2 + (玩家容貌 + 玩家气质) / 60 - 事件修正`

其中：

- 轻事件修正：`5`
- 中事件修正：`10`
- 重事件修正：`15`

最终成功率范围固定在 `10~90` 之间。

这个设计意味着：

- 宠爱高的人，皇帝更愿意听她解释
- 容貌和气质高的人，更容易引起怜惜
- 但重案始终更难洗清，不会因为漂亮就完全无敌

## 六、惩罚规则

## 6.1 基础惩罚等级

为了方便系统实现和后期调参，惩罚不再用一个大随机区间，而是先采用固定基准值：

- 轻事件基础惩罚：`150` 声望
- 中事件基础惩罚：`350` 声望
- 重事件基础惩罚：`750` 声望

若受害目标宠爱值：

- 大于 `60`，则惩罚上调一级
- 小于 `20`，则惩罚下调一级

## 6.2 最终声望扣除

`最终扣除声望 = max(0, 基础惩罚 - 减免值 + 加重值)`

其中：

`减免值 = 玩家宠爱 * 1.2 + 0.06 * (玩家容貌 + 玩家气质) + 合谋妃嫔宠爱 * 0.5`

`加重值 = 宫斗对象宠爱 * 0.7 + 对方阵营值`

`对方阵营值 = min(100, 0.25 * 宫斗对象交好妃子宠爱总值)`

为了避免高宠高颜角色把所有惩罚都洗到 0，加入一条硬限制：

- `减免值最多不超过基础惩罚的 70%`

这样既保留了“受宠又招人怜惜的人更容易被从轻发落”的感觉，也避免了后期滚雪球后完全不怕宫斗失败。

## 6.3 其他处罚

除扣除声望外，宫斗失败后的附加处罚建议如下：

- 轻事件定罪：仅扣声望
- 中事件定罪：扣声望，并可禁足 `1 月`
- 重事件定罪：扣声望，禁足 `1 月`，并视当前位分和结果决定是否降位

若扣除后的声望已经低于当前位分门槛，则自动触发降位判定。

此外，无论之前是否交好，只要双方发生直接宫斗并定性为敌对，双方关系立即转为交恶。后续若想恢复关系，必须额外通过事件重新修复。

## 七、B 版桌面测试

下面的测试不依赖代码，而是直接用规则公式进行样本推演，目的是验证这套规则是否会出现明显硬伤。

## 7.1 测试一：高心计玩家造谣轻事件

条件如下：

- 玩家心计 `780`
- 玩家宠爱 `60`
- 目标心计 `350`
- 目标宠爱 `25`
- 事件类型：造谣“奢侈浪费”，轻事件
- 不嫁祸，不合谋

计算：

- 造谣攻击值 `A = 780 / 10 = 78`
- 目标防御值 `D = 350 / 10 + 25 / 20 = 35 + 1 = 36`
- 行动成功率 `= 50 + 78 - 36 - 0 = 92`，按上限取 `90`
- 隐匿成功率 `= 50 + 780 / 12 - 350 / 15 + 60 / 8 - 5`
- 约为 `50 + 65 - 23 + 7 - 5 = 94`，按上限取 `85`

若隐匿失败并进入调查：

- 初始获罪率 `= 20 + (100 - 85)/2 + max(0, 25 - 20)/2 - 60/5`
- 约为 `20 + 7.5 + 2.5 - 12 = 18`
- 按下限取 `20`

三旬增长：

- 轻事件共增长 `8 * 3 = 24`
- 最终获罪率 `= 44`

结论：

这个案例说明，高心计高宠爱玩家在处理轻事件时，确实会很有优势，但即便如此，一旦真的暴露，也只是较低风险，而不是完全免疫。这是合理的。

## 7.2 测试二：中档玩家造谣中事件

条件如下：

- 玩家心计 `620`
- 玩家宠爱 `35`
- 目标心计 `600`
- 目标宠爱 `40`
- 事件类型：造谣“与人偷情”，中事件
- 不嫁祸，不合谋

计算：

- `A = 62`
- `D = 60 + 2 = 62`
- 行动成功率 `= 50 + 62 - 62 - 5 = 45`
- 隐匿成功率 `= 50 + 620/12 - 600/15 + 35/8 - 10`
- 约为 `50 + 51 - 40 + 4 - 10 = 55`

若隐匿失败并进入调查：

- 初始获罪率 `= 40 + (100 - 55)/2 + max(0, 40 - 20)/2 - 35/5`
- 约为 `40 + 22.5 + 10 - 7 = 65.5`
- 取整后约 `66`

三旬增长：

- 中事件共增长 `14 * 3 = 42`
- 最终获罪率 `= 108`

结论：

这一档是 B 版最关键的验证样本。结果显示，中档人物在做中事件时并不是完全不能玩，但一旦暴露，就必须花钱打点，否则很可能被定罪。这个风险水平是合适的。

## 7.3 测试三：弱势玩家造谣重事件

条件如下：

- 玩家心计 `520`
- 玩家宠爱 `20`
- 目标心计 `850`
- 目标宠爱 `75`
- 事件类型：造谣“意图谋逆”，重事件
- 不嫁祸，不合谋

计算：

- `A = 52`
- `D = 85 + 3 = 88`
- 行动成功率 `= 50 + 52 - 88 - 10 = 4`，按下限取 `10`
- 隐匿成功率 `= 50 + 520/12 - 850/15 + 20/8 - 15`
- 约为 `50 + 43 - 56 + 2 - 15 = 24`

若进入调查：

- 初始获罪率约为 `95`
- 三旬增长 `= 20 * 3 = 60`
- 最终获罪率 `= 155`

结论：

这个结果说明，低配玩家挑战高心计、高宠爱目标的重事件时，系统会给出非常明确的高风险反馈，不会让“不合理的冒险”轻易过关。这是合理的。

## 7.4 测试四：高药理玩家下毒轻事件

条件如下：

- 玩家药理 `85`
- 玩家心计 `700`
- 玩家宠爱 `50`
- 目标心计 `400`
- 目标宠爱 `30`
- 目标药理 `20`
- 事件类型：陨颜丹，轻事件

计算：

- `A = 85`
- `D = 400/10 + 30/20 + 20/4 = 40 + 1 + 5 = 46`
- 行动成功率 `= 50 + 85 - 46 = 89`
- 隐匿成功率 `= 50 + 700/12 - 400/15 + 50/8 - 5`
- 约为 `83`

若进入调查：

- 初始获罪率约 `24`
- 三旬增长后约 `48`

结论：

高药理玩家处理轻度下毒时确实较强，但并没有强到“必杀且无风险”。同时，这也能很好地支持药理路线的存在价值。

## 7.5 测试五：中档玩家下毒中事件

条件如下：

- 玩家药理 `68`
- 玩家心计 `620`
- 玩家宠爱 `35`
- 目标心计 `650`
- 目标宠爱 `55`
- 目标药理 `45`
- 事件类型：麝香，中事件

计算：

- `A = 68`
- `D = 65 + 2 + 11 = 78`
- 行动成功率 `= 50 + 68 - 78 - 5 = 35`
- 隐匿成功率约 `52`

若进入调查：

- 初始获罪率约 `75`
- 三旬增长后约 `117`

结论：

这说明中毒类中事件比普通造谣更危险，尤其是目标本身还懂药理时，会明显提高下毒难度和后续风险。这一点符合宫斗逻辑。

## 7.6 测试六：高药理玩家对高警觉目标使用重毒

条件如下：

- 玩家药理 `90`
- 玩家心计 `760`
- 玩家宠爱 `45`
- 目标心计 `750`
- 目标宠爱 `70`
- 目标药理 `80`
- 事件类型：鹤顶红，重事件

计算：

- `A = 90`
- `D = 75 + 3 + 20 = 98`
- 行动成功率 `= 50 + 90 - 98 - 10 = 32`
- 隐匿成功率约 `53`
- 初始获罪率通常会触顶到 `95`
- 三旬后约 `155`

结论：

这说明就算玩家药理很高，只要目标本身也强，而且事件本身足够严重，系统依然能维持高风险，不会出现后期数值成型后随意秒人的硬伤。

## 7.7 测试七：合谋收益测试

条件如下：

- 玩家心计 `600`
- 玩家宠爱 `30`
- 合谋妃嫔心计 `780`
- 合谋妃嫔宠爱 `50`
- 目标心计 `700`
- 目标宠爱 `40`
- 事件类型：中事件

不合谋时：

- 玩家攻击值 `A = 60`
- `D = 70 + 2 = 72`
- 行动成功率 `= 50 + 60 - 72 - 5 = 33`

合谋后：

- 新攻击值 `A = floor((60 + 78) / 2) + 10 = 79`
- 行动成功率 `= 50 + 79 - 72 - 5 = 52`
- 隐匿检定额外 `+5`

若立案：

- 初始获罪率会额外 `+10`

结论：

合谋显著提高了执行成功率，但并不是白送收益。它的作用是“更容易做成”，代价则是“被查时会更麻烦”。这条平衡是成立的。

## 7.8 测试八：嫁祸风险测试

仍以上一组条件为基础，只加入嫁祸。

结果变化：

- 行动检定额外 `-12`
- 隐匿检定额外 `-15`
- 初始获罪率额外 `+15`
- 每旬增长额外 `+5`

结论：

嫁祸会让整件事明显变得更危险，这样才能防止玩家把嫁祸当成无脑标配。保留嫁祸是好的，但必须像 B 版这样，把它写成一个“更贪但更险”的操作，而不是单纯多一个选择。

## 7.9 测试九：高宠爱对调查结果的影响

仍以测试五为基础，只替换玩家宠爱。

当玩家宠爱 `80` 时：

- 隐匿成功率会略微提升
- 初始获罪率会从约 `75` 降到约 `63`
- 三旬后约 `105`

当玩家宠爱 `20` 时：

- 初始获罪率约 `79`
- 三旬后约 `121`

结论：

高宠爱确实能帮助玩家压案，但不会直接变成无敌。这个结果比原版“100 - 宠爱”合理得多，因为原版会让高宠玩家在很多案件里天然无罪。

## 7.10 测试十：高宠爱目标的反制能力

仍以测试五为基础，只把目标宠爱提高到 `80`。

结果：

- 行动成功率略降
- 初始获罪率提升到约 `87`
- 三旬后约 `129`

结论：

被宠爱的目标更难碰，这是非常合理的。这样一来，玩家在选择目标时就必须考虑“值不值得冒这个险”，而不是看见谁不顺眼就直接下手。

## 7.11 测试十一：被诬陷后的自证

### 情况一：高宠高容貌高气质

- 玩家宠爱 `60`
- 容貌 `850`
- 气质 `800`
- 中事件

自证成功率：

`25 + 60/2 + (850 + 800)/60 - 10`

约为：

`25 + 30 + 27 - 10 = 72`

### 情况二：低宠中档容貌气质

- 玩家宠爱 `20`
- 容貌 `650`
- 气质 `650`
- 中事件

成功率约为：

`25 + 10 + 21 - 10 = 46`

### 情况三：低宠低数值重事件

- 玩家宠爱 `10`
- 容貌 `400`
- 气质 `450`
- 重事件

成功率约为：

`25 + 5 + 14 - 15 = 29`

结论：

这个公式很好地保留了你想要的“越招人怜惜，越容易让皇帝心软”的逻辑，但不会极端到只靠宠爱就能洗白一切。

## 七、测试后的正式优化结论

经过以上测试，B 版目前可以认为是三版里最适合优先落地的版本。原因如下：

第一，它去掉了原版最大的硬伤，也就是前置难度档位造成的断崖式概率。现在的系统是连续变化的，玩家不会再遇到“同样的角色，只差一点点属性，结果成功率从 64% 掉到 16%”这种非常割裂的体验。

第二，它把两次检定彻底分工了。第一次只管事件有没有做成，第二次只管有没有暴露。这样，系统结果可解释、可复盘、可写纪事，也更方便以后做 UI。

第三，它让高宠、高心计、高药理角色确实强，但没有强到无敌。高宠会帮助压案，高心计会帮助隐匿，高药理会帮助下毒，但每个属性的影响都是有边界的。

第四，它让合谋和嫁祸真正变成“风险换收益”的选择。合谋提高成功率，但被查时会更麻烦；嫁祸让玩家有更高的贪心空间，但也会显著提高翻车代价。这才像真正的宫斗系统。

第五，它的所有系数都比较容易调。将来如果你觉得某条路线太强，只需要调 `事件修正`、`初始获罪率基础值`、`每旬增长`、`宠爱减免系数` 这几处，不需要推翻整个系统。

## 八、最终建议

如果你现在要把这份内容单独写进“系统硬规则”，我建议直接采用这份 B 版定稿，不再回到最早那个“简单 / 一般 / 困难先分档，再决定两次都成功还是任意一次成功”的结构。

保留的内容应当是：

- 造谣与下毒分开
- 当旬夜晚统一结算
- 下一旬娇娇通报
- 可合谋
- 可嫁祸
- 最多三旬调查
- 打点压案
- 被诬陷时有单独自证检定

优化后的正式结论是：

- 难度标签仅作展示，不作前置门槛
- 两次检定分别对应“行动”和“隐匿”
- 调查完全由获罪率公式推进
- 宠爱、容貌、气质只影响“包庇和怜惜”，不再直接篡改事件真值

这套结构已经足够稳定，适合进入后续系统文档与实现阶段。

## 九、妃嫔之间互相宫斗的系统规则

上文的宫斗公式并不只适用于玩家对 NPC，也适用于妃嫔之间互相造谣、互相下手。为了保证系统统一、可追踪、可复盘，NPC 与 NPC 之间的宫斗不应另做一套隐藏逻辑，而应直接复用同一套 **行动检定 + 隐匿检定 + 调查获罪率** 的结构。

但如果直接按“任意两名交恶妃嫔，每旬都各自做一次 10% 判定”，随着后宫人数增加，宫斗事件会呈指数增长，很快把主线和玩家操作淹没。因此，这里必须加一个约束：**每名妃嫔每旬最多只能主动发起一次宫斗尝试**，而不是对每个仇敌都单独掷一轮概率。

### 9.1 NPC 主动宫斗的触发前提

某位 NPC 想要主动发起宫斗，必须同时满足以下条件：

- 与至少一名角色处于交恶状态
- 自身野心值大于 `50`
- 当前不在冷宫
- 当前未疯癫
- 当前未死亡
- 当前不处于本旬禁足状态
- 当前不处于本旬被重点调查状态

只要这些条件都成立，该 NPC 在每旬开始时，会获得一次主动宫斗判定资格。

### 9.2 NPC 主动宫斗触发概率

你提出的原始想法是：“如果二者交恶，且其中一方野心值大于 50，则每旬有 10% 概率进行宫斗。”  
这个思路可以保留，但必须改成 **按进攻者结算，而不是按敌对关系对结算**。

正式规则建议写为：

- 若 NPC 满足主动宫斗前提，则每旬有 `10%` 概率触发一次主动宫斗
- 这个 `10%` 只掷一次
- 一旦掷中，再从她当前所有交恶对象中选出目标

这样做的好处是：

- 后宫即使有很多交恶关系，也不会同旬爆出大量案件
- 有野心的妃子会显得更危险，但不会变成每旬都在乱斗
- 玩家能感受到“宫里有人在布局”，而不是系统无意义刷事件

### 9.3 NPC 目标选择

若 NPC 通过了主动宫斗触发，则从当前交恶对象中选择目标。目标不应随机平均分配，而应有一个简单权重，体现“高威胁目标更容易被针对”。

建议权重为：

`目标权重 = 20 + 目标宠爱 / 2 + 目标声望 / 100 + 是否怀孕加权 + 是否近期得宠加权`

其中：

- 若目标当前怀孕：`+15`
- 若目标最近两旬宠爱值增长明显：`+10`

若目标为玩家，则额外再加：

- 若玩家当前宠爱大于 `50`：`+10`
- 若玩家当前位分高于该 NPC：`+10`

这个权重的逻辑很明确：越受宠、越显眼、越挡路的人，越容易成为主动打击对象。

### 9.4 NPC 选择造谣还是下毒

NPC 不应完全随机选择行动类型，而应由属性偏好驱动。

建议规则为：

- 若 `心计换算值 >= 药理 + 15`，优先造谣
- 若 `药理 >= 心计换算值 + 15`，优先下毒
- 若两者接近，则：
  - 野心 `50~69`：`70%` 造谣，`30%` 下毒
  - 野心 `70~89`：`55%` 造谣，`45%` 下毒
  - 野心 `90+`：`40%` 造谣，`60%` 下毒

其中 `心计换算值 = 心计 / 10`。

这意味着：

- 心计型妃子更擅长玩名声战
- 药理型妃子更倾向暗手
- 野心极高的角色会更激进，也更容易做出高风险操作

### 9.5 NPC 事件轻重选择

NPC 不应一上来就频繁选择最重事件，否则后宫死亡率会过高，皇帝心情系统会被迅速打崩。

建议按野心值决定事件轻重分布：

- 野心 `51~69`
  - 轻事件：`60%`
  - 中事件：`35%`
  - 重事件：`5%`
- 野心 `70~89`
  - 轻事件：`35%`
  - 中事件：`45%`
  - 重事件：`20%`
- 野心 `90+`
  - 轻事件：`20%`
  - 中事件：`45%`
  - 重事件：`35%`

这样可以防止系统前期过度血腥，也能让高野心 NPC 在后期确实具备威胁性。

### 9.6 NPC 对 NPC 的公式复用

一旦进入正式宫斗，NPC 与 NPC 之间完全复用前文相同的公式：

- 行动检定
- 隐匿检定
- 调查获罪率
- 惩罚判定

区别只在于，玩家没有手动点条，而是系统直接计算。

因此，当某妃嫔对另一妃嫔发动造谣或下毒时，仍然按下列逻辑结算：

1. 计算攻击值与防御值
2. 计算行动成功率
3. 计算隐匿成功率
4. 得出四种结果之一
5. 若隐匿失败，则开启最多三旬调查
6. 到期结算定罪、惩罚、关系恶化、皇帝心情影响

### 9.7 NPC 与玩家相关的特殊处理

当 NPC 主动宫斗的目标是玩家时，玩家不能只是收到一个结算结果，而应进入一个强制交互流程。

因此，若玩家成为被害目标或被诬陷对象，系统在下一旬不只播报结果，还必须把玩家带入专门的 **养心殿事件**。

## 十、养心殿对质与自证清白规则

### 10.1 进入养心殿的触发条件

玩家会在以下情况下被强制带入养心殿界面：

- 玩家相关宫斗案件已有嫌疑人定案率达到 `100`
- 玩家主动宫斗后暴露，并被推至定罪门槛
- 玩家被其他妃嫔诬陷或嫁祸，并被推至定罪门槛
- 玩家成为被害妃嫔，且案件已进入待裁断

也就是说，玩家不是在调查一开始就被拖入养心殿，而是在案件已经达到定罪门槛、需要最终裁断时进入养心殿。

### 10.2 养心殿事件的系统定位

养心殿并不是重新做一轮完整案件系统，而是一个 **定罪前的强制裁断节点**。
它的作用是：

- 给玩家一个最终求情或认罚的场景
- 给受害妃嫔、被嫁祸者、定罪候选人和关键证人一个发言场景
- 让皇帝的宠爱、情绪、态度真正通过玩法体现出来
- 在不推翻前面公式的前提下，给玩家最后一次有限度地减轻惩罚的机会

因此，养心殿不重算“行动是否成功”，也不重算“是否隐匿失败”，只处理以下事项：

- 相关人发言
- 玩家选择求情姿态
- `最终惩罚修正`
- 定罪、处罚和关系变化写入

### 10.3 养心殿基础流程

进入养心殿后，第一版固定流程如下：

1. 下一旬清晨，玩家先在寝宫被内侍传旨
2. 玩家确认后被带到养心殿
3. 皇帝开场，说明案件内容
4. 定罪候选人、受害者、被嫁祸者、实际发起者、证人等按案件轻重出场发言
5. 玩家选择应对语气
6. 皇帝裁断，写入定罪与处罚
7. 裁断结束后仍停留在养心殿内，玩家自行选择外出或回宫

### 10.4 玩家应对语气的作用

玩家在养心殿不需要复杂多轮辩论，但按钮必须按玩家身份分流，不能把自辩、旁听和替人求情混成一套。

当玩家本人是定罪候选人时，按钮为：

- 据证自辩：减罚最多，但最容易激怒受害者一方
- 陈明疑点：指出案中疑处，中等减罚
- 伏身求宽：承认处境，请求从轻，稳定减罚
- 攀指旁人：把疑点引向旁人，可能减罚，但留下关系隐患
- 沉默领罪：不主动减罚，避免额外言语冲突

当玩家不是定罪候选人，而是受害者、被嫁祸者或相关旁听时，按钮为：

- 请皇上严断：加重处罚，受害者一方更认可
- 只陈案情：只补事实，不主动加重或减轻
- 指出疑点：小幅减轻处罚，关系波动较低
- 代为求情：替定罪候选人求宽，减罚较多，但得罪受害者一方
- 沉默旁听：不介入裁断，避免额外关系冲突

这样做的好处是，养心殿不只是一个过场，而是真正让玩家根据自己的 build 和当下处境做选择。

### 10.5 自证成功后的影响

若玩家在养心殿求情成功，不代表案件翻案，而是对最终处罚产生修正。

建议规则：

- 轻事件：可较明显减轻处罚
- 中事件：中等幅度减轻处罚
- 重事件：只能有限减轻处罚

当前实现采用“只减处罚，不翻案、不降定案率”的方式，等后面再扩展翻案或审判分支。

### 10.6 自证失败后的影响

若玩家自证失败，则：

- 维持原有获罪率
- 若是重事件，额外 `+10` 获罪率
- 与受害妃嫔关系直接转为交恶
- 若皇帝当前心情低于 `0`，玩家宠爱额外 `-5`

这会让养心殿真正具备压力，而不是进场走个过场。

### 10.7 被害妃嫔与玩家的交互意义

你特别提到：玩家在养心殿不仅要和皇帝交互，也要和被害妃嫔交互。这个设计是对的。

但在 V1 中，不建议把这部分再做成一套复杂分支，而是作为案件态度的额外修正源。

建议做法：

- 若被害妃嫔对玩家原本好感较高：
  - 她的发言会更克制
  - 玩家最终惩罚 `-5%`
- 若被害妃嫔对玩家本就交恶：
  - 她的发言更激烈
  - 玩家最终惩罚 `+5%`
- 若被害妃嫔宠爱 `> 60`：
  - 皇帝更偏向她
  - 玩家自证成功率额外 `-5`

这样就能让“和谁结仇”“对方受不受宠”真正反映在养心殿交互中。

## 十一、NPC 互斗接入现有文档后的整体结论

把 NPC 互斗与养心殿系统接进来以后，当前这套宫斗系统的整体闭环变成：

1. 玩家或 NPC 在白天形成宫斗行为
2. 当旬夜晚统一进行行动检定与隐匿检定
3. 下一旬由娇娇通报宫中消息
4. 若案件暴露，则进入最多三旬调查
5. 若玩家卷入案件核心，则强制进入养心殿事件
6. 玩家在养心殿与皇帝、受害妃嫔交互并做一次自证或请罪选择
7. 案件到期后统一结算获罪率、惩罚、关系、宠爱、声望与位分变化

这一闭环的优点是：

- 玩家行为与 NPC 行为共用同一套逻辑
- 系统结算清晰，AI 不参与真值
- 玩家在关键节点仍有互动权，不会只看系统播报
- 后宫会活起来，但不会因为 10% 互斗机制写法不当而同旬爆出大量事件

## 十二、当前版本需要额外明确的两条实现约束

为了避免后面实现时出现歧义，我建议把下面两条也直接写进硬规则。

### 12.1 每旬互斗上限

- 玩家每旬最多主动发动 `1` 次宫斗
- 每名 NPC 每旬最多主动发动 `1` 次宫斗
- 同一名角色同一旬内不能既是主动发起者又再次主动发起第二起案件

这样可以强行控制系统复杂度。

### 12.2 皇帝过问上限

不是所有案件都要把玩家拉去养心殿。  
只有玩家本人卷入，或者案件已经进入“皇帝亲自过问”的状态，才触发养心殿事件。

这样可以避免轻微 NPC 互斗也频繁切入主界面，影响节奏。
