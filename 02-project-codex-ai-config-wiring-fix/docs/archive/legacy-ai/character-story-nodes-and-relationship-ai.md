# 角色剧情节点与关系 AI 接口

## 一、文档定位

本文件单独负责整理：

- 固定妃子剧情节点
- 可攻略 NPC 剧情节点
- 皇帝容安在不同路线下的关键剧情节点
- 玩家与 NPC 互动时的关系加成 AI 接口

这份文档不负责真实数值结算。  
真实数值仍由硬规则系统负责。  
AI 在这里的职责只有两类：

- 生成剧情文本与角色反应
- 对玩家按钮选项做“关系倾向判定”

角色对白、场景礼法、称谓口径与人物说话方式，统一以 [character-dialogue-system-patch.md](/C:/02-project/docs/character-dialogue-system-patch.md) 为补充约束；若与角色主档冲突，以角色主档为准；若与硬规则冲突，以硬规则为准。

## 二、节奏原则

### 2.1 节点结构

每个重要角色至少应有 `3` 个层级节点：

1. 初识节点
2. 压力节点
3. 选择节点

若该角色可攻略，则再增加：

4. 倾情节点
5. 终局节点

### 2.2 默认前提

剧情设计默认玩家会：

- 主动攻略部分妃子
- 与部分 NPC 发展暧昧或偷情线
- 在多条关系中做取舍

因此剧情节点应主动制造：

- 关系冲突
- 立场反转
- 旧事暴露
- 在“皇帝 / 妃子 / NPC / 家族 / 故国”之间的拉扯

### 2.3 全局触发节奏规划

建议把剧情节点按四层推进：

| 阶段 | 时间建议 | 作用 |
| --- | --- | --- |
| 开场识别层 | 第 `1~3` 月 | 建立角色印象、确定关系基调 |
| 压力推进层 | 第 `4~8` 月 | 制造误会、争宠、站队压力 |
| 分支选择层 | 第 `9~18` 月 | 进入攻略、翻案、权力、逃离等中期分支 |
| 结局回收层 | 第 `19` 月后 | 根据路线、关系与关键节点进行回收 |

### 2.4 单月触发上限

为避免剧情过密，建议系统做以下限制：

- 每月重大角色节点最多 `2` 个
- 同一角色每月主节点最多 `1` 个
- 同一旬内“剧情节点 + 宫斗大事件 + 夜晚强制事件”合计不超过 `2` 个

### 2.5 触发条件类型

所有节点建议只用以下 6 类硬条件组合：

1. 时间条件：月 / 旬 / 年度
2. 关系条件：好感 / 倾情 / 交好 / 交恶
3. 路线条件：四条路线限定
4. 场景条件：御花园 / 养心殿 / 宝华殿 / 妙音堂 / 太医院等
5. 事件条件：宫斗成功、被陷害、怀孕、冷宫、翻案、避暑、选秀
6. 状态条件：位分、宠爱、真心、健康、政治、是否存活

### 2.6 无 AI 时的节点落地

即使完全关闭 AI，剧情节点也必须可跑。

因此每个节点都应至少拥有：

- `eventId`
- `triggerRule`
- `sceneId`
- `optionSetId`
- `resultPatch`
- `fallbackTextId`

这样在无 AI 模式下，也能靠模板文本和固定选项完成整条剧情推进。

## 三、固定妃子剧情节点

### 3.1 姚玲儿

基础定位：

- 兰因絮果核心对手
- 皇帝表妹
- 骄纵傲慢，但不是纯粹无脑反派

节点建议：

1. `宫宴失态`
   - 早期触发
   - 姚玲儿会在宫宴上对玩家的中宫地位明讽暗刺
   - 玩家可选择压下、回击或故作宽容

2. `表兄妹旧情`
   - 中期触发
   - 玩家撞见她与皇帝谈及旧日往事
   - 强化“青梅旧情”对玩家的压力

3. `利益结盟`
   - 若玩家与她关系改善，则她愿意与玩家共同压制其他妃嫔
   - 若长期交恶，则她会成为最稳定的高位竞争者

4. `醋意翻面`
   - 若其倾情升高且发现玩家多线攻略，则容易触发吃醋与试探

5. `谋逆协助`
   - 好感高、倾情高时可进入玩家权力线协助节点

### 3.2 柳仪芳

基础定位：

- 对玩家初始有好感
- 感恩自己因长相似玩家而获得如今生活
- 不会主动挑衅玩家

节点建议：

1. `感恩相见`
   - 初识时明确表达恭敬与感恩
   - 她会把自己当前安稳生活部分归因于“像玩家”

2. `替身风波`
   - 皇帝在侍寝、宫宴或偶遇场景中把她与玩家产生情感投射
   - 玩家可选择怜惜她、利用她、或疏远她

3. `自我选择`
   - 中期她会逐渐意识到自己不能永远做替身
   - 玩家若持续善待，她会更偏向站在玩家这一侧
   - 玩家若长期冷待，她会更依赖皇帝宠爱来保身，但仍不主动针对玩家

4. `争宠协助`
   - 倾情较高时，她会主动替玩家争取衣饰、曲谱、侍寝前的消息

5. `替身破局`
   - 后期可触发“她不再模仿玩家，而是主动做自己”的成长节点

### 3.3 江晚晚

基础定位：

- 东宫旧人
- 温柔知性
- 真实立场最难看透

节点建议：

1. `东宫旧物`
   - 玩家偶然得到旧物，引出她与皇帝在东宫时期的旧事

2. `旧情与旧伤`
   - 她会在某个节点透露皇帝最忌惮或最放不下的往事
   - 玩家可借此拉近她，也可因此更戒备她

3. `递册与藏册`
   - 她掌握某本册录、书信或旧年名册
   - 可成为宫斗、翻案、立储线的重要信息中转人

4. `温柔偏刀`
   - 若玩家过度信任，她可能在关键节点保持沉默而不是明害
   - 这类“看似无害、实际不帮”的反差会增强她的压迫感

5. `站队节点`
   - 后期必须在玩家、皇帝与自保之间站队

### 3.4 沈妙清

基础定位：

- 浮生如梦的情感锚点
- 玩家旧友
- 年纪小，容易被卷入后宫斗争

节点建议：

1. `旧友重逢`
   - 入宫初期，她会是玩家最温和的情绪缓冲点

2. `被迫长大`
   - 中期因受欺压、被挑拨或被利用而快速成长

3. `连带受伤`
   - 若玩家主动宫斗频繁，她更容易被卷入流言与报复

4. `要不要保护她`
   - 玩家必须在“保护她的纯真”和“把她培养成盟友”之间做选择

5. `感情转折`
   - 倾情上升后，她会从依赖玩家转成希望独占玩家关注

### 3.5 陈婉宁

基础定位：

- 影落掖庭核心对手兼关键翻案枢纽
- 与玩家家族旧案直接相关

节点建议：

1. `笑里藏刀`
   - 初期表面示好，暗中试探玩家是否仍执着翻案

2. `交易条件`
   - 她会抛出条件：若玩家放弃某部分追查，她可减轻打压

3. `证物节点`
   - 她掌握或接近掌握旧案关键证物
   - 玩家可偷取、交易、威逼、攻略

4. `情与利翻转`
   - 倾情高时，她会在感情和家族利益之间摇摆

5. `翻案协助或最终阻击`
   - 后期若未攻略成功，她将成为翻案线最强阻力
   - 若攻略成功，则会反过来替玩家撬开旧案

## 四、可攻略 NPC 剧情节点

### 4.1 当一

基础定位：

- 佛子线
- 禁欲、克制、带救赎感

节点建议：

1. `佛前问心`
2. `破戒边缘`
3. `假死协助`
4. `留或不留`

重点冲突：

- 情与戒律
- 救人还是守道

### 4.2 简宁

基础定位：

- 太医令
- 外冷内热
- 医者仁心

节点建议：

1. `戒备诊脉`
2. `药理教习`
3. `宫人疫病 / 义诊节点`
4. `假死药抉择`

重点冲突：

- 医者原则与对玩家的私情

### 4.3 卢安平

基础定位：

- 青梅竹马哥哥
- 稳重博学
- 朝堂与皇嗣线的重要外援

节点建议：

1. `旧物来信`
2. `为孩子辅导与季度汇报`
3. `朝堂牵线`
4. `他知道你不止爱他`
5. `留在身边或主动离开`

重点冲突：

- 他能给玩家最稳的帮助，但也最容易因长期得不到回应而心灰

### 4.4 布自游

基础定位：

- 御膳房线
- 阳光、热烈、生活气

节点建议：

1. `热食与闲话`
2. `御膳房小秘密`
3. `节庆大菜`
4. `只此一次的越界相助`

重点冲突：

- 他提供的是“活着的烟火气”，适合成为玩家高压宫斗中的情绪休息点

### 4.5 连翘

基础定位：

- 琴师线
- 气质成长与曲谱系统核心 NPC

节点建议：

1. `妙音堂初见`
2. `赠谱节点`
3. `共演节点`
4. `借曲传情`

重点冲突：

- 她不是强戏剧冲突型角色，更适合做“慢热知己”与才艺成长线的承接者

### 4.6 阿翎

基础定位：

- 尘缘夙错专属
- 故国青梅竹马
- 外部世界与逃离可能性的具象化

节点建议：

1. `第三个月宫门重逢`
2. `离宫邀请`
3. `年度书信`
4. `第4年夜闯宫门 / 战死 / 劝皇帝善待玩家` 三分支

重点冲突：

- 她代表“宫外的自由”和“没有来得及实现的旧情”

## 五、皇帝容安的分路线节点

### 5.1 兰因絮果

定位：

- 中宫夫妻 / 权力伴侣

建议节点：

1. `中宫共治`
2. `表妹与皇后的权衡`
3. `选秀与六宫决断`
4. `你是妻子还是政治答案`

### 5.2 浮生如梦

定位：

- 从低位走向被看见

建议节点：

1. `偶遇带回养心殿`
2. `第一次真正求情成功`
3. `他开始在你面前显露疲惫`
4. `他把选择权递给你`

### 5.3 影落掖庭

定位：

- 从被压制到翻案

建议节点：

1. `他最初并不真正相信你`
2. `旧案证物与质问`
3. `他要的是安稳还是真相`
4. `翻案后你还愿不愿意留在他身边`

### 5.4 尘缘夙错

定位：

- 和亲、异族、真心与权力冲突

建议节点：

1. `他把你当成和书的一部分`
2. `他开始把你当成活人`
3. `故国、阿翎与皇帝的三角拉扯`
4. `你要他的爱、他的皇位，还是离开他`

## 六、关系 AI 接口

### 6.1 定位

该接口单独存在，不用于文本补全。  
它只负责判断：玩家本次按钮选项，在关系上更偏向“友好、暧昧、冷淡、拒斥”中的哪一种。

### 6.2 推荐接口名

- `relationship-judge`

### 6.3 输入

| 字段 | 说明 |
| --- | --- |
| `routeId` | 当前路线 |
| `npcId` | 当前交互对象 |
| `sceneType` | 场景类型，如拜访、偶遇、寝殿谈心、御花园、养心殿等 |
| `optionText` | 玩家点击的按钮文本 |
| `npcProfile` | 角色性格摘要 |
| `currentFavor` | 当前好感 |
| `currentAffection` | 当前倾情 |
| `recentContext` | 最近数次交互摘要 |

### 6.4 输出

| 字段 | 说明 |
| --- | --- |
| `toneTag` | `friendly / flirt / cold / reject / neutral` |
| `favorDelta` | `-1 / 0 / 1` |
| `affectionDelta` | `-1 / 0 / 1` |
| `reason` | 简短解释 |
| `confidence` | 置信度 |

### 6.5 系统映射

- 若 AI 判定为 `friendly`，则本次 `好感 +1`
- 若 AI 判定为 `flirt`，则本次 `倾情 +1`
- 若 AI 判定为 `cold`，则本次 `好感 -1`
- 若 AI 判定为 `reject`，则本次 `倾情 -1`
- `neutral` 默认 `+0`
- 单次按钮只允许命中一种关系变化，不允许同时改动好感与倾情

### 6.6 每旬上限

每旬每个 NPC 最多：

- `好感变化绝对值 <= 5`
- `倾情变化绝对值 <= 5`

超过上限后，即使 AI 继续给出判断，系统也不再继续改动该旬对应该角色的关系值。

### 6.7 特殊事件硬覆盖

以下情况不走关系 AI 小幅加减，而直接套用硬规则：

- 若玩家确认陷害某角色，且事件被该角色确认或结案落地：
  - 轻事件：该角色对玩家好感至多保留到 `0`
  - 中事件：该角色对玩家好感直接压到 `-10`
  - 重事件：该角色对玩家好感直接压到 `-20`
- 若玩家对某角色实施下毒并落地：
  - 默认按重事件处理
- 这条规则优先级高于普通对话关系变化

### 6.8 边界

- 该接口不负责生成对白
- 该接口不负责真实剧情结果
- 该接口不允许直接修改除好感/倾情外的其他数值
- 最终加成仍由系统做上限校验后落地

### 6.9 无 AI 降级方案

若关系 AI 不可用，则系统直接读取按钮配置中的本地标签：

- `friendly`
- `flirt`
- `cold`
- `reject`
- `neutral`

并按同一映射规则结算。  
这样即使第二个 AI 完全关闭，游戏也仍可完整推进。

## 七、双 AI 与节点落地建议

正式开发时，建议把这份文档拆成两部分：

1. `character-story-node-config`
2. `relationship-ai-judge`
3. `narrative-text-ai`

这样好处很直接：

- 剧情节点可以继续扩写
- 关系 AI 可以独立接入，不污染主文本 AI
- 文本 AI 与关系 AI 彻底分离，形成双 AI 结构
- 后续固定妃子、自定义剧情妃、可攻略 NPC 都能复用同一套关系判断逻辑
