# 后宫游戏系统硬规则总稿

## 一、使用说明

本文件是当前阶段的**总链路硬规则主稿**。  
后续正式编写游戏代码时，应优先以本文件为主；若与其他旧稿存在冲突，以本文件为准。

系统原则只有一条：

- 所有真实结果由固定数值系统判定
- AI 只负责文本表现、对话包装和情绪反馈

### 1.1 双 AI 原则

本项目预留两个相互独立的 AI 接口：

1. `narrative-text-ai`
2. `relationship-judge-ai`

职责划分：

- `narrative-text-ai`：负责剧情文本、对白润色、通报、角色口吻与事件包装
- `relationship-judge-ai`：负责判断玩家按钮选项更偏“友好 / 暧昧 / 冷淡 / 拒斥”，并返回建议的好感 / 倾情微调

两者都不得直接决定硬结果。

### 1.2 无 AI 运行要求

即使两个 AI 全部关闭，游戏也必须可完整跑通。

无 AI 模式下：

- 当前版本不接入文本 AI 正文；正常玩法只从本地 CSV 剧情表、事件标题和固定系统文案取文本
- 关系 AI 由按钮本地标签替代，系统直接读取 `friendly / flirt / cold / reject / neutral`
- 所有结局、位分、怀孕、宫斗、案件、冷宫、家族、皇嗣结果仍全部可算出

因此 AI 只允许做“增强层”，绝不能成为推进主流程的必要条件。

### 1.3 剧情文本配置原则

- 剧情正文、说话人、基础立绘元数据和场景提示维护在 `src/game/narrative/csv/`；当前版本 CSV 是正常玩法唯一剧情正文源。
- CSV 按系统域拆分，不按“一段剧情一个文件”拆，也不合并为单一大表。
- 代码只引用稳定剧情 ID；需要剧情演出时应按完整 entry 消费说话人、立绘、旁白名和正文，不能只把 `text` 抽到 CSV。
- 业务 runtime 必须通过 `narrativeDialogueAdapter` 将 CSV entry 转换为开场响应、全局舞台字段、妃嫔会面回合或普通展示字段；不得在各流程中重复逐项读取 `entry.speakerName`、`entry.text`、`entry.portraitKey` 来拼展示对象。
- CSV 不写 JS 表达式、不写数值公式、不写条件判断；下一步按钮、纯 UI 标签、短错误提示可以继续留在代码中，但不得作为剧情 turn / AI response / CSV 字段传递。
- CSV 正文中的显式分页继续使用 `<<PAGE_BREAK>>`；换行写为 `\n` 字符串，不使用真实跨行单元格。
- 新增剧情时必须同步补充 CSV loader / catalog 引用测试，确保重复 ID、空正文、非法立绘位置和缺失 ID 能被发现；当前玩法代码不得重新引入 `WithFallback` 类剧情入口或第二正文来源。

### 1.4 数值配置原则

- 核心可调数值维护在 `src/game/numerics/csv/`，由 `numericCatalog` 统一解析、校验和导出。
- CSV 只承载参数、初始数据和策划说明，不写 JS 表达式、随机公式或状态流转。范围统一使用 `min` / `max` 两列，多值列表使用 `|`。
- 当前已表格化的核心域包括：属性字段、全局范围与倍率、路线初始、路线固定属性、寝殿行动、月用度、宠爱分层、位分声望表、库存物品、绣花 / 字画 / 调香作品、固定妃嫔种子数值、随机补足妃嫔模板与生成浮动、宫斗严重度 / 裁断档位、夜晚侍寝池 / 兴致结果档位 / 第三方影响参数。
- 如果需要把公式从业务 runtime 中抽出，必须以完整公式维护在 `src/game/numerics/formula-pages/*FormulaPage.ts`，并写清注释；运行期通过 `formulaRuntime.ts` 的受限解析器求值。公式页只能放公式和说明，不混入解析器、状态写入或业务分支。不得把公式拆成 `base`、`divisor`、`bonus` 等半截 CSV 行。
- 旧模块可以保留导出名作为兼容层，但真实值必须从 `numericCatalog` 读取，不能在业务代码里继续维护第二套同名核心数值。
- 存档不是 CSV。`SaveGameV1` 的 schema version、storage key、必需字段和默认进度块维护在 `src/game/save/saveGameConfig.ts`；真实玩家存档仍保存在 localStorage envelope 中。
- 当前处于开发期，存档结构不兼容时直接清除旧存档，不做旧结构 fallback 或迁移。

## 二、主循环

### 2.1 单旬循环

1. 旬开始
2. 自由行动
3. 夜晚判定
4. 深夜收尾
5. 进入下一旬清晨，并按次日口径重算体力

### 2.2 月循环

每月末统一结算：

- 皇帝心情恢复
- 月俸与宫务
- 子嗣抚养
- 家族与朝堂
- 怀孕推进
- 位分推进
- 冷宫判定

## 三、时间与体力

- 每旬 `7` 个时间格
- 体力上限 `15`
- 每旬初始体力 `10`
- 旬末进入下一旬清晨时，体力不继承旧值，而是按“新旬基础体力 + 状态修正”重新计算
- 当前未接入额外修正时，重算结果固定为 `10`
- 体力为 `0` 时不能进行常规行动
- 进出场景、打开地点面板、从外景回地图只改变所在位置，不消耗体力，也不推进时辰；只有地点内明确行动才消耗时间格。

### 3.1 寝殿固定行动数值

| 行为 | 时间格 | 体力 | 固定效果 |
| --- | --- | --- | --- |
| 诵读经典 | `1` | `-2` | `poetry +1`，`politics +1` |
| 泼墨作画 | `1` | `-3` | `painting +1`，若选择进行中作品则推进字画进度 |
| 习舞奏乐 | `1` | `-2` | `temperament +1`，`talent +1`（显示为乐理） |
| 镂月裁云 | `1` | `-3` | `embroidery +1`（显示为刺绣），若选择进行中作品则推进绣花进度 |
| 调制香薰 | `1` | `-3` | `appearance +1`，若选择进行中作品则推进调香进度 |
| 请平安脉 | `1` | `0` | `health +5`，每旬只可请一次 |
| 殿内小憩 | `1` | `+3` | `stress -1` |
| 外出探索 | `0` | `0` | 切至地图，不直接改数值 |
| 结束本旬 | `0` | `0` | 推进到下一旬清晨并重算体力 |

### 3.2 作品制作

- 绣花、字画、调香作品只能通过寝殿对应行动进入，不设独立“作品管理”总入口。
- 点击 `镂月裁云 / 泼墨作画 / 调制香薰` 后，先进入该类别制作面板；玩家可添加该类别作品或选择进行中作品推进。当前不提供“仅练习”分支。
- 作品题材、主 / 辅能力、难度、基础售价和基础送礼好感维护在 `src/game/numerics/csv/craft_works.csv`。
- 单次进度、完成成色、最终售价和送礼好感使用 `src/game/numerics/formula-pages/craftWorkFormulaPage.ts` 的完整公式，不写入 CSV。
- 完成品进入背包 `gift` 分类，可以作为礼物赠送，也可以变卖换银两。调香第一版不产生毒药、药效或特殊状态。

## 四、属性主轴

- 声望：位分主轴
- 宠爱：夜晚权重、经济加成、案件减伤
- 福德：怀孕主轴
- 野心：结局与 NPC 主动宫斗倾向
- 心计：造谣与隐匿核心
- 药理：下毒核心
- 容貌 / 气质：侍寝兴致、求情宽判、心软系表现
- 健康：生产结果、死亡风险
- 政治：朝堂与家族后期核心

### 4.1 后期容貌 / 气质成长

- 习舞奏乐：`-2` 体力，`1` 时间格，`乐理 +2`，`气质 +3`，`容貌 +3`
- 御花园闲逛：`-1` 体力，`1` 时间格，`气质 +3`
- 华清池温泉：`-1` 体力，`1` 时间格，`容貌 +3`

这几项属于稳定成长项，保证后期容貌与气质存在持续上升路径。

### 4.2 妃嫔旬级行动

- 每旬为 live 且非冷宫妃嫔生成一次 NPC 主行动。
- 主行动包括留宫、公共外出、拜访玩家、拜访其他妃嫔、社交筹谋、敌意筹谋。
- 公共外出地点以地图热点配置为准，空地图点进入后也可出现 NPC 交互入口；不得只在几个功能小场景中局部随机抓人。
- NPC 拜访其他妃嫔时，被拜访者原本外出取消并留殿接待，玩家进入被拜访者宫殿时能看到两人同场。
- NPC 拜访玩家必须带目的，并在寝殿空闲时自动打断流程；NPC 先发起小对话，玩家必须选择接待、试探或拒绝。
- 公共地点遇见 NPC 时不得自动进入妃嫔对话；应先显示可点击的妃嫔交互入口，玩家主动点击后再用地点入场叙述和行动摘要进入妃嫔日常对话。
- 地图地点弹窗只展示地点自身描述，不得展示本旬 NPC 名字、摘要或动向，也不得新增“前去见某妃”直达选项；玩家进入地点后才能点击妃嫔交互入口。公共外出的 `resolved` 只表示本旬交谈已触发，不代表 NPC 离开目的地。
- 妃嫔会面是强剧情顺序：入场对白未收起时不得点击固定操作；最后一次互动必须先播放本次互动结果，再播放送客 / 让路收束。殿内收束用宫人送客与“下旬再叙”，公共地点偶遇收束用随侍提醒行程与“今日偶遇到此为止”。
- 地图“后宫”热点不得额外提供“后宫总览”快捷选项；玩家统一通过 `进入此处` 进入后宫外景。
- 后宫外景打开的左侧工具面板必须记住后宫返回点；关闭回后宫，面板内 `外出` 直接回地图主视角，不得中转玩家寝宫。
- 外景回地图时不得立即清理 `activeMapLocation`；旧外景位置和面板状态必须保留到退出动画完成后再清理，避免旧场景在淡出期间按寝宫状态重绘。
- 被其他 NPC 拜访且尚未收束的妃嫔必须在自己寝宫标记为“会客中”。
- 未收束的 `visit-consort` 目标妃嫔以会客状态为本旬位置真值；旧公共外出条目不得让她从寝宫消失，也不得让她同时出现在公共地点可交互名单中。玩家结束殿内会客后，`visit-consort.resolved=true` 表示来访者回自己的寝宫，目标不再“会客中”。
- `旧居X` 类型住址按 `X` 主殿参与后宫宫殿匹配。
- NPC-NPC 关系结算造成的压力、紧张、好感等内部变化默认静默写入状态和关系矩阵；除非形成案件、通报或玩家亲眼参与，否则不得用玩家 toast 直接告知。
- NPC 宫斗每旬全局只掷一次，默认 `10%`，每旬最多生成一条 NPC 宫斗案。

## 五、月俸与经济

### 5.1 月俸

- 按位分领取月俸
- `位分 < 妃` 且 `宠爱 1~19`：`70%`
- `位分 < 妃` 且 `宠爱 <= 0`：`50%`
- 冷宫：`0`
- `宠爱 >= 61`：月俸 `+20%`

### 5.2 倾情赠银

- `倾情 > 60`
- 每月最多 `2` 名
- 每名赠自己当月实际月俸的 `1/4`

### 5.3 月度用度

设当月实际月俸为 `S`：

```txt
节衣缩食 = floor(S * 0.25)，声望 -5，健康 -1
量入为出 = floor(S * 0.5)，声望 0，健康 0
锦衣玉食 = floor(S * 0.75)，声望 +10，健康 +1
```

开局三选项只决定本月用度策略；寝殿“调整用度”登记下月策略。

### 5.4 礼物、补品与珍稀度

| 品质 | 颜色 | 绣品 / 字画 / 补品价格 | 送礼好感 | 补品健康效果 | 补品容貌效果 |
| --- | --- | --- | --- | --- | --- |
| 绿色 | Green | `70` | `+5` | `+20` | `+5` |
| 蓝色 | Blue | `130` | `+10` | `+40` | `+10` |
| 紫色 | Purple | `180` | `+15` | `+60` | `+15` |
| 红色 | Red | `250` | `+20` | `+80` | `+20` |

规则：

- 绣品、字画按品质增加目标好感。
- 补品可送礼，也可自用。
- 补品送礼时同时增加目标好感、健康与容貌。
- 补品自用时恢复健康与容貌，不增加好感。
- 御膳房珍馐：`60`
- 稀有礼物：`260`
- 特殊礼物：`420`

### 5.5 杜娘稀有道具

| 物品 | 单价 | 效果 |
| --- | --- | --- |
| 冷香丸 | `500` | `气质 +100` |
| 驻颜丹 | `500` | `容貌 +100` |
| 延寿丹 | `500` | `寿命 +5` |
| 缠梦香 | `1000` | 赠予目标后，每旬额外 `寿命 -0.6`，可叠加 |

规则：

- 杜娘每次刷新时，每种稀有道具独立 `40%` 概率出现。
- 同一种稀有道具每次最多出现 `2` 个。
- 若玩家 `药理 > 80`，可花费 `500` 自制缠梦香，每次 `70%` 概率成功。
- 缠梦香不进入常规赏赐池。

### 5.6 赏赐与特殊赠礼

- `倾情 >= 80` 的妃子在首次达到阈值时，会赠送玩家 `冷香丸 / 驻颜丹` 其中之一。
- `宠爱 >= 61` 的妃子每月有 `60%` 概率获得皇帝赏赐。
- `太后好感 >= 61` 的妃子每月有 `60%` 概率获得太后赏赐。
- 皇帝赏赐池中不出现 `延寿丹 / 缠梦香`。
- 太后赏赐池中不出现 `冷香丸 / 驻颜丹 / 延寿丹 / 缠梦香`。

### 5.7 子嗣支出

- 0~35月：每名子嗣每月 `20`
- 36月以上：每名子嗣每月 `35`
- 满月礼：`180`
- 周岁礼：`260`
- 两岁礼：`320`

### 5.8 年度节庆

协理六宫 / 皇贵妃 / 皇后每年一次：

- 支出 `260`
- 成功：
  - 其他妃子野心 `-4`
  - 玩家压力 `-10`
  - 玩家野心 `+6`
  - 玩家声望 `+20`

## 六、夜晚系统

### 6.1 玩家优先检定

夜晚先检定玩家，再轮到其他妃嫔。

### 6.2 夜晚加成

- 御书房：`+15`
- 御花园偶遇：`+10`
- 宝华殿为皇帝祈福：`+10`
- 若玩家为真心值最高对象：额外 `+15`
- 每旬未侍寝保底默认 `+2`，真实值由 `nightly_runtime_rules.csv` 的 `player_night_favor_gauge_increment` 维护。

### 6.3 独寝

皇帝独寝率由心情决定，默认档位维护在 `nightly_emperor_alone_rates.csv`：

- 悲痛 `50%`
- 悲伤 `35%`
- 低落 `25%`
- 平常 `15%`
- 愉悦 `10%`
- 喜悦 `5%`

### 6.4 侍寝兴致

- 最低默认 `20`，最高默认 `100`，由 `nightly_runtime_rules.csv` 维护。
- 第二次初始兴致 = 上次兴致的一半
- 盛宠及以上底值 `60`

### 6.5 侍寝中的第三方操作

- 玩家可美言交好妃子：目标宠爱默认 `+3~5`
- 玩家可抹黑交恶妃子：目标宠爱默认 `-3~5`
- 交好妃子侍寝：默认 `80%` 概率为玩家美言
- 交好妃子为玩家美言时，玩家除获得宠爱变化外，还会获得声望；位格越高，声望增量越高
- 交恶妃子侍寝：默认 `20%` 概率抹黑玩家
- 玩家被传召侍寝：默认固定声望 `+10`
- 侍寝表现声望：按 `nightly_interest_effects.csv` 的兴致档位额外结算

以上默认值由 `nightly_runtime_rules.csv` 与 `nightly_interest_effects.csv` 维护。

以上均不算主动宫斗，不扣福德，不进调查。

## 七、怀孕与生育

### 7.1 怀孕

- 每次侍寝后按 `福德值 = 本次怀孕概率`
- 若福德 `< 0`：直接 `0%`
- 怀孕后次日早晨直接显性
- 怀孕期间无法侍寝，但可进入陪伴池
- 怀孕期间每旬 `+10` 声望

### 7.2 生育冷却

- 生产后：`3` 个月
- 流产后：`3` 个月

### 7.3 流产

- 流产统一 `健康 -60`
- 若麝香导致：
  - `30%` 永久不孕
  - `10%` 一尸两命

永久不孕统一表现为：

- `福德 = -99`
- 后续不会再被改变

### 7.4 宝华殿与福德恢复

- 宝华殿为自己生育祈福：每次 `福德 +3`
- 整月未主动宫斗：月末 `福德 +5`

### 7.5 宝华殿寿命预警

- 若宫中任一被追踪角色的自然剩余寿命 `< 24` 个月，则玩家进入宝华殿时会触发一次“冥冥之音”提示。
- 提示对象包括：玩家自己、皇帝、太后、妃子、全部可攻略角色。
- 系统默认优先提示剩余寿命最短的目标。
- 该提示不直接显示精确寿命数值，只给出“今年似有油尽灯枯之兆”等文本警示。

### 7.6 生产结果

- 健康 `> 400`：正常生育
- 健康 `201~400`
  - `5%` 死胎
  - 否则 `30%` 难产，生子后健康 `-50`
- 健康 `1~200`
  - `10%` 死胎，健康 `-60`，容貌 `-120`，气质 `-120`
  - 否则 `50%` 难产，健康 `-50`，容貌 `-120`

## 八、宫斗系统

### 8.1 类型

- 造谣
- 下毒

玩家主动下毒必须先在背包中持有对应毒药。当前毒药入口收束到掖庭院“月姑姑”交易：陨颜丹、麝香、鹤顶红。投放 QTE 成功并登记案件时消耗 `1` 份；QTE 失败不建案、不消耗。

### 8.2 主动宫斗的福德代价

- 造谣 `-5`
- 下毒 `-10`

### 8.3 特殊关系硬覆盖

若玩家对某角色实施陷害或下毒，并且事件被该角色确认或结案落地，则不走普通对话加减，而直接按严重度覆盖其对玩家的好感：

- 轻事件：好感至多保留到 `0`
- 中事件：好感压到 `-10`
- 重事件：好感压到 `-20`

这条是硬规则，优先级高于普通关系变化。

### 8.4 公式

- 造谣攻击值：`心计 / 10`
- 下毒攻击值：`药理`
- 目标防御值：
  - 造谣：`目标心计 / 10 + 目标宠爱 / 20`
  - 下毒：`目标心计 / 10 + 目标宠爱 / 20 + 目标药理 / 4`

### 8.5 两次检定

1. 行动检定
2. 隐匿检定

### 8.6 合谋与嫁祸

- 好感 `> 60` 可合谋
- 嫁祸提高风险

## 九、案件与嫌疑

### 9.1 嫌疑人结构

暴露案件必须生成最多三名嫌疑人。嫌疑人字段包括主体类型、主体 ID、名称、定案率、是否实际发起者、是否被嫁祸和嫌疑理由。旧 `convictionRate` 只同步最高定案率展示，不再作为唯一裁判。

嫌疑人生成优先级：

- 实际发起者
- 被嫁祸者；初始定案率至少 `70`
- 玩家本人，如果玩家是被害者或被嫁祸者
- 其余从存活非冷宫妃嫔中按动机补足

### 9.2 调查期

- 最多 `3` 旬
- 每旬所有嫌疑人同步增长
- 轻 / 中 / 重案件分别 `+8 / +14 / +20`
- 被嫁祸者每旬额外 `+5`
- 任一嫌疑人定案率达到 `100`，案件进入 `pending_verdict` 待裁断，不立即扣声望 / 宠爱 / 压力
- 三旬无人达到 `100`，归档为疑案

### 9.3 宫斗事务调查页

- 调查入口在“宫斗事务”内，有调查中或已归档案件时才显示
- `20` 银两：任一嫌疑人定案率 `-5`
- `20` 银两：任一嫌疑人定案率 `+5`；到 `100` 后进入待裁断
- 以上干预可重复操作，直到待裁断、结案或银两不足

## 十、皇帝系统

### 10.1 心情月恢复

- `mood < 0`：月末 `+8`
- `mood >= 0`：月末 `+3`

### 10.2 真心值

影响：

- 夜晚权重
- 御书房
- 养心殿说服
- 真爱值最高保护

### 10.3 皇帝日间行为与养心殿

- 清晨：皇帝在正阳门。玩家进入正阳门后可选择“等待下朝”，按概率偶遇皇帝，成功宠爱 `+1`。
- 上午：皇帝在养心殿处理政务。玩家可求见，但成功率低于中午 / 下午 / 傍晚，不是固定拒绝。
- 中午 / 下午 / 傍晚：皇帝通常在养心殿，也可能出现在御花园或建章宫；若玩家进入皇帝所在公共地点，可点击皇帝入口进入完整互动。
- 夜晚 / 深夜：养心殿不接受后宫求见，玩家进入后由内侍劝归。
- 地点进入本身不消耗时间格和体力；`activeMapLocationEntryTime` 只用于记录入场时辰，求见、等待下朝、公共偶遇和皇帝互动是否耗时由各自明确行动规则决定。
- 皇帝交互每次只允许 `1` 次主行动，另可在送礼 / 美言 / 诉苦中选 `1` 次附加话题。
- 主行动按相关养成属性、玩家宠爱、真心值和皇帝心情判定平淡 / 小成 / 大成；真实数值影响限定在宠爱、声望、真心。
- 皇帝公共地点偶遇必须先播放主行动结果，再播放外景恭送收束；不得把“恭送圣驾”提前覆盖行动结果，也不得主行动后直接黑屏离开。
- 送礼消耗真实背包食物、绣品或字画；美言使目标妃嫔声望 `+5`，诉苦使目标妃嫔声望 `-5`。
- 养心殿裁断仍只负责自证、求情和对惩罚做有限修正；不加入此前测试阶段使用过的高位额外幽禁强规则。

## 十五、自定义剧情妃与 AI 接口

### 15.1 基本原则

- 玩家可通过召见娇娇提交自定义剧情妃草案。
- 系统负责字段校验、入宫合法性与最终数值落地。
- AI 只负责草案解析、次旬通报、人设文本与后续剧情对白。

### 15.2 入宫流程

1. 玩家在寝殿召见娇娇
2. 选择 `引荐新妃 / 新增剧情妃`
3. 用结构化表单或自然语言提交性格、家世、资质
4. 系统写入 `pendingCustomConsort`
5. 下一旬由娇娇通报其以“家族举荐 / 和亲 / 外邦进献 / 宫女抬位”等合理方式入宫
6. 新角色正式接入夜晚、宫斗、怀孕、位分、家族、皇嗣等全部系统

### 15.3 字段原则

- 自定义剧情妃共享普通妃嫔的全部系统字段
- 额外增加：`isCustomConsort`、`entrySource`、`personaCard`、`insertedAtXun`
- 自定义模式默认不强制总点数平衡，只校验单字段上下限

### 15.4 推荐接口

- `custom-consort-draft`
- `custom-consort-intro`
- `custom-consort-dialogue`

详细字段与流程以《自定义剧情妃与 AI 接口规则》为准。

### 15.5 关系 AI 上限

普通对话导致的关系变化，默认遵守：

- 每旬每妃嫔普通互动回合数 `<= 3`
- 每旬每角色 `好感变化绝对值 <= 5`
- 每旬每角色 `倾情变化绝对值 <= 5`
- 单次按钮只改动一个维度
- 妃嫔会面中单个话题最多继续 `2` 句，后续必须收束或切换行动
- 殿内主动会面上限触顶时不得只做按钮禁用，必须有合理化剧情演出：送客、婉拒、通传或时间压力，并明确玩家为何本旬不能继续交互

只有宫斗、下毒、结案、救命、替罪、舍命相救等特殊事件可绕开该限制。

新增机制时也必须遵守同一原则：硬规则需要对应剧情解释和演出效果，不能只把数值或按钮状态裸露给玩家。

### 15.6 尘缘夙错专属 NPC：阿翎

阿翎为 `尘缘夙错` 路线专属固定女性 NPC，年龄 `22`，定位为故国青梅竹马。

触发流程：

1. 第 `3` 个月，阿翎出现在宫门处，由娇娇前来通报。
2. 玩家可前往对话并开启长期支线。
3. 当阿翎好感 `> 80` 时，她会询问玩家是否愿意和她一起离开。
4. 若玩家当时拒绝，阿翎会留下：
   - `冷香丸 x2`
   - `驻颜丹 x2`
   - `延寿丹 x1`
   - `缠梦香 x1`
   - 离别信件 `x1`

书信推进：

- 之后每年 `12` 月清晨，娇娇都会转达阿翎来信。
- 离开后的第 `1~2` 年，信件内容以游历见闻与思念为主。
- 第 `3` 年，阿翎会坦白自己自幼倾心于玩家，并表达“想成为女将军保护你”的心意。

第 `4` 年分支：

- 若玩家 `宠爱 < 30`：
  - 阿翎回宫夜闯宫门，邀玩家离开
  - 玩家若拒绝，阿翎死亡
  - 玩家若同意，触发“回故国并收复故土”的双人幸福结局
- 若玩家 `30 < 宠爱 < 60`：
  - 次月由娇娇通报：阿翎已在故国成为将军，并与皇帝容安开战后身死
  - 皇帝会在御书房召见玩家，说明阿翎死前仍念玩家姓名
  - 事件后玩家 `宠爱 +10`
- 若玩家 `宠爱 > 60`：
  - 阿翎写信劝皇帝善待玩家，不再开战
  - 事件后玩家 `宠爱 +10`，皇帝 `真心值 +10`
  - 自此不再来信，但往后每年 `12` 月固定收到 `冷香丸 x1` 与 `驻颜丹 x1`

## 十六、正式开发前的确认清单

正式开始编码前，应先确认以下 8 条：

1. 去掉 AI 后，所有系统仍可用模板文本与本地标签完整推进
2. 双 AI 接口都只做增强，不直接写硬结果
3. 每旬每角色普通对话导致的好感 / 倾情变化不超过 `5`
4. 宫斗与下毒会触发关系硬覆盖，不走普通对话增减
5. 所有剧情节点都能映射为 `eventId + triggerRule + optionSetId + resultPatch`
6. 自定义剧情妃通过 `pending -> next xun insert` 正式接入
7. 四条路线都能在无 AI 模式下完成从开局到结局的流程
8. 所有 AI 返回值都必须经过系统二次校验后再落地

## 十七、正式开发建议

正式开始写代码时，建议按这 7 个模块分层：

1. `core-state`
2. `time-and-month-settlement`
3. `night-and-pregnancy`
4. `palace-strife-and-cases`
5. `rank-and-governance`
6. `family-court-and-heirs`
7. `custom-consort-and-ai-bridge`

这样当前整条链路就能直接落到：

- store
- config
- settlement functions
- event scheduler
- AI prompt inputs

而不需要再回到设计层反复改口径。
