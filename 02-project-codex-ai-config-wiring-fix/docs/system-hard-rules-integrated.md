# 后宫游戏系统硬规则总稿

## 一、使用说明

本文件是当前阶段的**总链路硬规则主稿**。  
后续正式编写游戏代码时，应优先以本文件为主；若与其他旧稿存在冲突，以本文件为准。

系统原则只有一条：

- 所有真实结果由固定数值系统判定
- AI 只负责文本表现、对话包装和情绪反馈

### 1.1 双 AI 原则

本项目预留两个相互独立的 AI 接口：

1. `narrative-text-ai`
2. `relationship-judge-ai`

职责划分：

- `narrative-text-ai`：负责剧情文本、对白润色、通报、角色口吻与事件包装
- `relationship-judge-ai`：负责判断玩家按钮选项更偏“友好 / 暧昧 / 冷淡 / 拒斥”，并返回建议的好感 / 倾情微调

两者都不得直接决定硬结果。

### 1.2 无 AI 运行要求

即使两个 AI 全部关闭，游戏也必须可完整跑通。

无 AI 模式下：

- 文本 AI 由模板文本库、事件标题、固定对白表替代
- 关系 AI 由按钮本地标签替代，系统直接读取 `friendly / flirt / cold / reject / neutral`
- 所有结局、位分、怀孕、宫斗、案件、冷宫、家族、皇嗣结果仍全部可算出

因此 AI 只允许做“增强层”，绝不能成为推进主流程的必要条件。

## 二、主循环

### 2.1 单旬循环

1. 旬开始
2. 自由行动
3. 夜晚判定
4. 深夜收尾
5. 进入下一旬清晨，并按次日口径重算体力

### 2.2 月循环

每月末统一结算：

- 皇帝心情恢复
- 月俸与宫务
- 子嗣抚养
- 家族与朝堂
- 怀孕推进
- 位分推进
- 冷宫判定

## 三、时间与体力

- 每旬 `7` 个时间格
- 体力上限 `15`
- 每旬初始体力 `10`
- 旬末进入下一旬清晨时，体力不继承旧值，而是按“新旬基础体力 + 状态修正”重新计算
- 当前未接入额外修正时，重算结果固定为 `10`
- 体力为 `0` 时不能进行常规行动

### 3.1 寝殿固定行动数值

| 行为 | 时间格 | 体力 | 固定效果 |
| --- | --- | --- | --- |
| 诵读经典 | `1` | `-1` | `poetry +2` |
| 泼墨作画 | `1` | `-1` | `painting +2` |
| 习舞奏乐 | `1` | `-2` | `talent +2`（显示为乐理），`temperament +3`，`appearance +3` |
| 镂月裁云 | `1` | `-1` | `embroidery +2` |
| 调制香薰 | `1` | `-1` | `medicine +2` |
| 请平安脉 | `1` | `0` | `health +3`，`stress -1` |
| 殿内小酣 | `1` | `+3` | 仅恢复体力 |
| 外出探索 | `0` | `0` | 切至地图，不直接改数值 |
| 结束本旬 | `0` | `0` | 推进到下一旬清晨并重算体力 |

## 四、属性主轴

- 声望：位分主轴
- 宠爱：夜晚权重、经济加成、案件减伤
- 福德：怀孕主轴
- 野心：结局与 NPC 主动宫斗倾向
- 心计：造谣与隐匿核心
- 药理：下毒核心
- 容貌 / 气质：侍寝兴致、求情宽判、心软系表现
- 健康：生产结果、死亡风险
- 政治：朝堂与家族后期核心

### 4.1 后期容貌 / 气质成长

- 习舞奏乐：`-2` 体力，`1` 时间格，`乐理 +2`，`气质 +3`，`容貌 +3`
- 御花园闲逛：`-1` 体力，`1` 时间格，`气质 +3`
- 华清池温泉：`-1` 体力，`1` 时间格，`容貌 +3`

这几项属于稳定成长项，保证后期容貌与气质存在持续上升路径。

### 4.2 妃嫔旬级行动

- 每旬为 live 且非冷宫妃嫔生成一次 NPC 主行动。
- 主行动包括留宫、公共外出、拜访玩家、拜访其他妃嫔、社交筹谋、敌意筹谋。
- 公共外出地点以地图热点配置为准，空地图点进入后也可出现 NPC 交互入口；不得只在几个功能小场景中局部随机抓人。
- NPC 拜访其他妃嫔时，被拜访者原本外出取消并留殿接待，玩家进入被拜访者宫殿时能看到两人同场。
- NPC 拜访玩家必须带目的，并在寝殿空闲时自动打断流程；NPC 先发起小对话，玩家必须选择接待、试探或拒绝。
- 公共地点遇见 NPC 时不得自动进入妃嫔对话；应先显示可点击的妃嫔交互入口，玩家主动点击后再用地点入场叙述和行动摘要进入妃嫔日常对话。
- 地图地点弹窗只展示地点自身描述，不得展示本旬 NPC 名字、摘要或动向，也不得新增“前去见某妃”直达选项；玩家进入地点后才能点击妃嫔交互入口。公共外出的 `resolved` 只表示本旬交谈已触发，不代表 NPC 离开目的地。
- 地图“后宫”热点不得额外提供“后宫总览”快捷选项；玩家统一通过 `进入此处` 进入后宫外景。
- 后宫外景打开的左侧工具面板必须记住后宫返回点；关闭回后宫，面板内 `外出` 直接回地图主视角，不得中转玩家寝宫。
- 外景回地图时不得立即清理 `activeMapLocation`；旧外景位置和面板状态必须保留到退出动画完成后再清理，避免旧场景在淡出期间按寝宫状态重绘。
- 被其他 NPC 拜访且尚未收束的妃嫔必须在自己寝宫标记为“会客中”。
- 未收束的 `visit-consort` 目标妃嫔以会客状态为本旬位置真值；旧公共外出条目不得让她从寝宫消失，也不得让她同时出现在公共地点可交互名单中。玩家结束殿内会客后，`visit-consort.resolved=true` 表示来访者回自己的寝宫，目标不再“会客中”。
- `旧居X` 类型住址按 `X` 主殿参与后宫宫殿匹配。
- NPC-NPC 关系结算造成的压力、紧张、好感等内部变化默认静默写入状态和关系矩阵；除非形成案件、通报或玩家亲眼参与，否则不得用玩家 toast 直接告知。
- NPC 宫斗每旬全局只掷一次，默认 `10%`，每旬最多生成一条 NPC 宫斗案。

## 五、月俸与经济

### 5.1 月俸

- 按位分领取月俸
- `位分 < 妃` 且 `宠爱 1~19`：`70%`
- `位分 < 妃` 且 `宠爱 <= 0`：`50%`
- 冷宫：`0`
- `宠爱 >= 61`：月俸 `+20%`

### 5.2 倾情赠银

- `倾情 > 60`
- 每月最多 `2` 名
- 每名赠自己当月实际月俸的 `1/4`

### 5.3 月度用度

设当月实际月俸为 `S`：

```txt
节衣缩食 = floor(S * 0.25)，声望 -5，健康 -1
量入为出 = floor(S * 0.5)，声望 0，健康 0
锦衣玉食 = floor(S * 0.75)，声望 +10，健康 +1
```

开局三选项只决定本月用度策略；寝殿“调整用度”登记下月策略。

### 5.4 礼物、补品与珍稀度

| 品质 | 颜色 | 绣品 / 字画 / 补品价格 | 送礼好感 | 补品健康效果 | 补品容貌效果 |
| --- | --- | --- | --- | --- | --- |
| 绿色 | Green | `70` | `+5` | `+20` | `+5` |
| 蓝色 | Blue | `130` | `+10` | `+40` | `+10` |
| 紫色 | Purple | `180` | `+15` | `+60` | `+15` |
| 红色 | Red | `250` | `+20` | `+80` | `+20` |

规则：

- 绣品、字画按品质增加目标好感。
- 补品可送礼，也可自用。
- 补品送礼时同时增加目标好感、健康与容貌。
- 补品自用时恢复健康与容貌，不增加好感。
- 御膳房珍馐：`60`
- 稀有礼物：`260`
- 特殊礼物：`420`

### 5.5 杜娘稀有道具

| 物品 | 单价 | 效果 |
| --- | --- | --- |
| 冷香丸 | `500` | `气质 +100` |
| 驻颜丹 | `500` | `容貌 +100` |
| 延寿丹 | `500` | `寿命 +5` |
| 缠梦香 | `1000` | 赠予目标后，每旬额外 `寿命 -0.6`，可叠加 |

规则：

- 杜娘每次刷新时，每种稀有道具独立 `40%` 概率出现。
- 同一种稀有道具每次最多出现 `2` 个。
- 若玩家 `药理 > 80`，可花费 `500` 自制缠梦香，每次 `70%` 概率成功。
- 缠梦香不进入常规赏赐池。

### 5.6 赏赐与特殊赠礼

- `倾情 >= 80` 的妃子在首次达到阈值时，会赠送玩家 `冷香丸 / 驻颜丹` 其中之一。
- `宠爱 >= 61` 的妃子每月有 `60%` 概率获得皇帝赏赐。
- `太后好感 >= 61` 的妃子每月有 `60%` 概率获得太后赏赐。
- 皇帝赏赐池中不出现 `延寿丹 / 缠梦香`。
- 太后赏赐池中不出现 `冷香丸 / 驻颜丹 / 延寿丹 / 缠梦香`。

### 5.7 子嗣支出

- 0~35月：每名子嗣每月 `20`
- 36月以上：每名子嗣每月 `35`
- 满月礼：`180`
- 周岁礼：`260`
- 两岁礼：`320`

### 5.8 年度节庆

协理六宫 / 皇贵妃 / 皇后每年一次：

- 支出 `260`
- 成功：
  - 其他妃子野心 `-4`
  - 玩家压力 `-10`
  - 玩家野心 `+6`
  - 玩家声望 `+20`

## 六、夜晚系统

### 6.1 玩家优先检定

夜晚先检定玩家，再轮到其他妃嫔。

### 6.2 夜晚加成

- 御书房：`+15`
- 御花园偶遇：`+10`
- 宝华殿为皇帝祈福：`+10`
- 若玩家为真心值最高对象：额外 `+15`
- 每旬未侍寝保底 `+2`

### 6.3 独寝

皇帝独寝率由心情决定：

- 悲痛 `50%`
- 悲伤 `35%`
- 低落 `25%`
- 平常 `15%`
- 愉悦 `10%`
- 喜悦 `5%`

### 6.4 侍寝兴致

- 最低 `20`
- 最高 `100`
- 第二次初始兴致 = 上次兴致的一半
- 盛宠及以上底值 `60`

### 6.5 侍寝中的第三方操作

- 玩家可美言交好妃子：目标宠爱 `+3~5`
- 玩家可抹黑交恶妃子：目标宠爱 `-3~5`
- 交好妃子侍寝：`80%` 概率为玩家美言
- 交恶妃子侍寝：`20%` 概率抹黑玩家
- 兴致 `100`：额外声望 `+10`

以上均不算主动宫斗，不扣福德，不进调查。

## 七、怀孕与生育

### 7.1 怀孕

- 每次侍寝后按 `福德值 = 本次怀孕概率`
- 若福德 `< 0`：直接 `0%`
- 怀孕后次日早晨直接显性
- 怀孕期间无法侍寝，但可进入陪伴池
- 怀孕期间每旬 `+10` 声望

### 7.2 生育冷却

- 生产后：`3` 个月
- 流产后：`3` 个月

### 7.3 流产

- 流产统一 `健康 -60`
- 若麝香导致：
  - `30%` 永久不孕
  - `10%` 一尸两命

永久不孕统一表现为：

- `福德 = -99`
- 后续不会再被改变

### 7.4 宝华殿与福德恢复

- 宝华殿为自己生育祈福：每次 `福德 +3`
- 整月未主动宫斗：月末 `福德 +5`

### 7.5 宝华殿寿命预警

- 若宫中任一被追踪角色的自然剩余寿命 `< 24` 个月，则玩家进入宝华殿时会触发一次“冥冥之音”提示。
- 提示对象包括：玩家自己、皇帝、太后、妃子、全部可攻略角色。
- 系统默认优先提示剩余寿命最短的目标。
- 该提示不直接显示精确寿命数值，只给出“今年似有油尽灯枯之兆”等文本警示。

### 7.6 生产结果

- 健康 `> 400`：正常生育
- 健康 `201~400`
  - `5%` 死胎
  - 否则 `30%` 难产，生子后健康 `-50`
- 健康 `1~200`
  - `10%` 死胎，健康 `-60`，容貌 `-120`，气质 `-120`
  - 否则 `50%` 难产，健康 `-50`，容貌 `-120`

## 八、宫斗系统

### 8.1 类型

- 造谣
- 下毒

### 8.2 主动宫斗的福德代价

- 造谣 `-5`
- 下毒 `-10`

### 8.3 特殊关系硬覆盖

若玩家对某角色实施陷害或下毒，并且事件被该角色确认或结案落地，则不走普通对话加减，而直接按严重度覆盖其对玩家的好感：

- 轻事件：好感至多保留到 `0`
- 中事件：好感压到 `-10`
- 重事件：好感压到 `-20`

这条是硬规则，优先级高于普通关系变化。

### 8.4 公式

- 造谣攻击值：`心计 / 10`
- 下毒攻击值：`药理`
- 目标防御值：
  - 造谣：`目标心计 / 10 + 目标宠爱 / 20`
  - 下毒：`目标心计 / 10 + 目标宠爱 / 20 + 目标药理 / 4`

### 8.5 两次检定

1. 行动检定
2. 隐匿检定

### 8.6 合谋与嫁祸

- 好感 `> 60` 可合谋
- 嫁祸提高风险

## 九、案件与嫌疑

### 9.1 个人事务

- `20` 银两：自身嫌疑 `-5`
- 若已锁定陷害妃嫔：
  - `20` 银两：该妃嫔嫌疑 `+5`
- 若该妃嫔嫌疑到 `100`：直接定罪
- 以上干预可重复操作，直到结案或银两不足

### 9.2 旁观案件

- `20` 银两：某妃嫔嫌疑 `-5`
- `20` 银两：某妃嫔嫌疑 `+5`
- 同样允许重复操作

### 9.3 调查期

- 最多 `3` 旬
- 个人案件打点仍按 `20 银两 -> 5 点变化`

## 十、皇帝系统

### 10.1 心情月恢复

- `mood < 0`：月末 `+8`
- `mood >= 0`：月末 `+3`

### 10.2 真心值

影响：

- 夜晚权重
- 御书房
- 养心殿说服
- 真爱值最高保护

### 10.3 养心殿

养心殿只负责：

- 自证
- 求情
- 对惩罚做有限修正

不再加入此前测试阶段使用过的高位额外幽禁强规则。

## 十一、位分与冷宫

### 11.1 位分推进

- 声望决定应有位分
- 自然晋升每月最多 `2` 级
- 自然降位每月最多 `2` 级

### 11.2 冷宫

只有两条路径：

1. 事件导致声望 `< 0`
2. 自然声望连续 `2` 月 `< -50`

### 11.3 真爱值最高保护

处罚后声望保底 `100`。  
仅适用于处罚事件。

## 十二、皇贵妃与协理六宫

### 12.1 皇贵妃开放

满足其一：

- 无皇后
- 皇后健康 `< 400` 持续 `3` 月
- 皇后宠爱 `< 20` 持续 `6` 月

皇后恢复后，已在位皇贵妃不自动撤位。

### 12.2 协理六宫

- 同时最多 `1` 人
- 每 `3` 个月一次调位
- 每次最多 `2` 级

## 十三、家族与朝堂

### 13.1 父亲官职月度加成

- 0级：`+0`
- 1级：`+2`
- 2级：`+4`
- 3级：`+6`
- 4级：`+8`
- 5级：`+10`

### 13.2 自然声望修正

- 商贾之女：`-3`
- 罪臣之后：`-5`
- 六品：`0`
- 四品：`+1`
- 二三品 / 一品：`+2`

### 13.3 前六个月保护

入宫前 `6` 个月，父亲官职不会降位。

### 13.4 家族接济

当前版本只保留第一档：

- `120` 银两

### 13.5 朝臣打点

- `政治 < 20`：不可正式打点

### 13.6 后期权力事件

- `2000 / 4000 / 6000` 三档

- 投入越高，朝堂衷心度、政治、真心值与夺位成功率越高

## 十五、自定义剧情妃与 AI 接口

### 15.1 基本原则

- 玩家可通过召见娇娇提交自定义剧情妃草案。
- 系统负责字段校验、入宫合法性与最终数值落地。
- AI 只负责草案解析、次旬通报、人设文本与后续剧情对白。

### 15.2 入宫流程

1. 玩家在寝殿召见娇娇
2. 选择 `引荐新妃 / 新增剧情妃`
3. 用结构化表单或自然语言提交性格、家世、资质
4. 系统写入 `pendingCustomConsort`
5. 下一旬由娇娇通报其以“家族举荐 / 和亲 / 外邦进献 / 宫女抬位”等合理方式入宫
6. 新角色正式接入夜晚、宫斗、怀孕、位分、家族、皇嗣等全部系统

### 15.3 字段原则

- 自定义剧情妃共享普通妃嫔的全部系统字段
- 额外增加：`isCustomConsort`、`entrySource`、`personaCard`、`insertedAtXun`
- 自定义模式默认不强制总点数平衡，只校验单字段上下限

### 15.4 推荐接口

- `custom-consort-draft`
- `custom-consort-intro`
- `custom-consort-dialogue`

详细字段与流程以《自定义剧情妃与 AI 接口规则》为准。

### 15.5 关系 AI 上限

普通对话导致的关系变化，默认遵守：

- 每旬每角色 `好感变化绝对值 <= 5`
- 每旬每角色 `倾情变化绝对值 <= 5`
- 单次按钮只改动一个维度

只有宫斗、下毒、结案、救命、替罪、舍命相救等特殊事件可绕开该限制。

### 15.6 尘缘夙错专属 NPC：阿翎

阿翎为 `尘缘夙错` 路线专属固定女性 NPC，年龄 `22`，定位为故国青梅竹马。

触发流程：

1. 第 `3` 个月，阿翎出现在宫门处，由娇娇前来通报。
2. 玩家可前往对话并开启长期支线。
3. 当阿翎好感 `> 80` 时，她会询问玩家是否愿意和她一起离开。
4. 若玩家当时拒绝，阿翎会留下：
   - `冷香丸 x2`
   - `驻颜丹 x2`
   - `延寿丹 x1`
   - `缠梦香 x1`
   - 离别信件 `x1`

书信推进：

- 之后每年 `12` 月清晨，娇娇都会转达阿翎来信。
- 离开后的第 `1~2` 年，信件内容以游历见闻与思念为主。
- 第 `3` 年，阿翎会坦白自己自幼倾心于玩家，并表达“想成为女将军保护你”的心意。

第 `4` 年分支：

- 若玩家 `宠爱 < 30`：
  - 阿翎回宫夜闯宫门，邀玩家离开
  - 玩家若拒绝，阿翎死亡
  - 玩家若同意，触发“回故国并收复故土”的双人幸福结局
- 若玩家 `30 < 宠爱 < 60`：
  - 次月由娇娇通报：阿翎已在故国成为将军，并与皇帝容安开战后身死
  - 皇帝会在御书房召见玩家，说明阿翎死前仍念玩家姓名
  - 事件后玩家 `宠爱 +10`
- 若玩家 `宠爱 > 60`：
  - 阿翎写信劝皇帝善待玩家，不再开战
  - 事件后玩家 `宠爱 +10`，皇帝 `真心值 +10`
  - 自此不再来信，但往后每年 `12` 月固定收到 `冷香丸 x1` 与 `驻颜丹 x1`

## 十六、正式开发前的确认清单

正式开始编码前，应先确认以下 8 条：

1. 去掉 AI 后，所有系统仍可用模板文本与本地标签完整推进
2. 双 AI 接口都只做增强，不直接写硬结果
3. 每旬每角色普通对话导致的好感 / 倾情变化不超过 `5`
4. 宫斗与下毒会触发关系硬覆盖，不走普通对话增减
5. 所有剧情节点都能映射为 `eventId + triggerRule + optionSetId + resultPatch`
6. 自定义剧情妃通过 `pending -> next xun insert` 正式接入
7. 四条路线都能在无 AI 模式下完成从开局到结局的流程
8. 所有 AI 返回值都必须经过系统二次校验后再落地

## 十七、正式开发建议

正式开始写代码时，建议按这 7 个模块分层：

1. `core-state`
2. `time-and-month-settlement`
3. `night-and-pregnancy`
4. `palace-strife-and-cases`
5. `rank-and-governance`
6. `family-court-and-heirs`
7. `custom-consort-and-ai-bridge`

这样当前整条链路就能直接落到：

- store
- config
- settlement functions
- event scheduler
- AI prompt inputs

而不需要再回到设计层反复改口径。
