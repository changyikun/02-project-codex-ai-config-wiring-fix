# 皇嗣管理与生育后续硬规则（架构目录版）

## 一、文档定位

本文件用于定义“怀孕后续、生育结果、皇嗣属性、早期养成、重华宫教育、朝臣支持、皇帝对子嗣宠爱”这一整条系统的硬规则。  
这部分规则完全由系统判定，不交给 AI 决定真实结果。AI 仅负责补充太医问诊、孩子成长对话、卢安平汇报、皇帝与玩家之间的文本表现。

本文件默认建立在前两份文档基础上：

- 《侍寝与怀孕硬规则》
- 《宫斗事务硬规则》

它们共同组成一条完整闭环：

夜晚互动 -> 怀孕判定 -> 孕期推进 -> 流产或生产 -> 子嗣生成 -> 皇嗣管理 -> 教育成长 -> 朝臣支持 -> 皇帝对子嗣宠爱变化

若玩家开启偷情线，则该闭环还需额外维护孩子的公开血脉与真实血脉字段；真实血脉不交给 AI 推断，而由系统直接记录。

## 二、已确认的前置规则

在本文件中，以下规则视为已定稿，不再重复争论：

1. 每次侍寝后，按 `福德值 = 本次怀孕概率` 进行怀孕判定
2. 若福德 `< 0`，则怀孕概率直接为 `0`
3. 怀孕成功后，第二天早上直接显性，由太医问诊并告知皇帝
4. 怀孕期间角色无法侍寝，但可进入“陪伴”夜晚互动
5. 怀孕期间每旬额外增加 `10` 声望
6. 若发生流产，则统一追加 `健康 -60`
7. 若孕期成功被下麝香，则直接流产，并额外判定：
   - `30%` 概率以后不孕不育，记为 `permanentInfertility = true`
   - `10%` 概率一尸两命
8. 流产若由宫斗导致，则后续调查、获罪率与皇帝心情处理依照宫斗事务规则
9. 宠爱不会因为流产本身直接波动
10. 若福德仍大于 `0`，则流产不影响后续继续怀孕
11. 生产后与流产后，统一进入 `3` 个月生育冷却期

## 三、生产结果正式定稿

生产结果完全按生产时母体健康值决定，不再附加其他隐藏项。

### 3.1 健康值 `> 400`

- 正常生育
- 母子平安
- 创建对应皇嗣记录
- 生育冷却 `3` 个月

### 3.2 健康值 `201 ~ 400`

按顺序判定：

1. 先判死胎：
   - 概率 `5%`
   - 若触发：产下死胎，母体健康 `-60`
2. 若未触发死胎，再判难产：
   - 概率 `30%`
   - 若触发：难产，生子后母体健康 `-50`
   - 同时创建对应皇嗣记录
3. 若两者都未触发：
   - 正常生育
   - 创建对应皇嗣记录

无论是本档难产还是正常生产，只要成功生育，都统一追加：

- 生育冷却 `3` 个月

### 3.3 健康值 `1 ~ 200`

按顺序判定：

1. 先判死胎：
   - 概率 `10%`
   - 若触发：产下死胎，母体健康 `-60`，容貌 `-120`，气质 `-120`
2. 若未触发死胎，再判难产：
   - 概率 `50%`
   - 若触发：难产，生子后母体健康 `-50`，容貌 `-120`
   - 创建对应皇嗣记录
3. 若两者都未触发：
   - 正常生育
   - 创建对应皇嗣记录

无论是本档难产还是正常生产，只要成功生育，都统一追加：

- 生育冷却 `3` 个月

### 3.4 流产后的皇帝心情

若为自然流产，则皇帝心情下降按“中等负面事件”处理。  
若为宫斗导致的流产，则皇帝心情下降完全按宫斗事务规则执行。  
若触发一尸两命，则额外叠加“死亡事件”对应的皇帝心情下降。

无论自然流产还是宫斗流产，统一追加：

- 生育冷却 `3` 个月

## 四、皇嗣生成时的基础属性

孩子出生后，进入“皇嗣管理”系统。为了方便快速实现和测试，V1 仅保留以下核心属性：

- `天赋值`
- `宠爱值`（皇帝对子嗣的宠爱）
- `健康`
- `容貌`
- `亲近`（对玩家的亲近度）
- `性格`
- `朝臣支持`
- `publicFatherId`
- `biologicalFatherId`
- `isSecretLineage`

### 4.1 属性取值范围

- 天赋值：`0 ~ 100`
- 宠爱值：`0 ~ 100`
- 健康：`0 ~ 100`
- 容貌：`0 ~ 1000`
- 亲近：`0 ~ 100`
- 朝臣支持：`0 ~ 100`

### 4.1.1 双父字段

若孩子来自正常侍寝，则：

- `publicFatherId = emperor`
- `biologicalFatherId = emperor`
- `isSecretLineage = false`

若孩子来自偷情线，则：

- `publicFatherId = emperor`
- `biologicalFatherId = 偷情对象`
- `isSecretLineage = true`

也就是说：

- 对外登记始终按皇帝血脉处理
- 系统内部仍保留真实生父字段
- 后续若触发“父子相认”剧情，只读取 `biologicalFatherId`

这就是偷情系统所需的双父结构。

### 4.2 性格池

性格为出生时随机生成，只从以下六类中取一项：

- 温和
- 腼腆
- 活泼
- 圆滑
- 高冷
- 傲娇

### 4.3 容貌

为了快速跑通，V1 直接采用母体快照值：

`子嗣容貌 = 生母生产时的容貌值`

这样最稳定，也最容易实现。

### 4.4 天赋值

你已经明确提出：天赋值取决于母亲心计高低和随机数。

推荐正式公式：

`子嗣天赋 = clamp(20, 95, 20 + floor(生母心计 / 20) + 随机值)`

其中：

- 随机值范围：`0 ~ 20`

这个公式的效果是：

- 生母心计 `200`，则子嗣天赋通常落在 `30 ~ 50`
- 生母心计 `600`，则子嗣天赋通常落在 `50 ~ 70`
- 生母心计 `900`，则子嗣天赋通常落在 `65 ~ 85`

这样既保留了“母亲越聪明，孩子越有天赋”的感觉，也保留了随机性。

### 4.5 健康

推荐正式公式：

`子嗣健康 = clamp(20, 100, 40 + floor(生母生产时健康 / 20) - 生产惩罚)`

其中：

- 正常生育：生产惩罚 `0`
- 难产：生产惩罚 `10`

若死胎，则不生成皇嗣记录。

### 4.6 亲近

亲近指孩子对玩家的亲近度。

为了快速跑通，推荐如下：

- 若孩子为玩家亲生：
  - 初始亲近固定 `60`
- 若孩子并非玩家亲生，但因后续剧情被纳入玩家可接触范围：
  - 初始亲近固定 `20`

V1 先不把亲近和太多复杂因素绑死，否则早期很难调数值。

### 4.7 朝臣支持初值

孩子刚出生时不直接拥有高额朝臣支持。  
初始支持主要由生母位分和皇帝对生母宠爱决定。

推荐公式：

`初始朝臣支持 = min(30, 生母位分支持 + floor(max(0, 生母宠爱) / 15))`

其中生母位分支持取固定档位：

- 皇后 / 皇贵妃：`20`
- 贵妃 / 四妃 / 妃：`15`
- 嫔 / 容华 / 婕妤 / 贵嫔：`10`
- 贵人 / 美人 / 才人 / 常在及以下：`5`

### 4.8 皇帝对子嗣的初始宠爱

你已经明确提出：孩子宠爱值取决于孩子的天赋、容貌和皇帝对于母亲的宠爱。

推荐正式公式：

`子嗣初始宠爱 = clamp(0, 100, 10 + floor(子嗣天赋 * 0.4) + floor(子嗣容貌 / 100) + floor(max(0, 生母宠爱) / 4) + 皇帝心情修正)`

其中皇帝心情修正：

- `mood < 0`：`-5`
- `1 ~ 50`：`0`
- `51 ~ 100`：`+5`

这个公式的好处是：

- 母亲越受宠，皇帝对孩子越容易生出偏爱
- 天赋高、容貌高的孩子更吃香
- 但不会直接因为母亲得宠就无脑冲到满值

## 五、性格在 V1 中的作用

为了快速实现，V1 不把性格做成大规模分支系统，只给少量成长修正。

### 温和

- 亲近类增长额外 `+1`

### 腼腆

- 初始子嗣宠爱 `-5`
- 亲近类增长额外 `+1`

### 活泼

- 健康成长类收益额外 `+1`

### 圆滑

- 朝臣支持类增长额外 `+1`

### 高冷

- 天赋成长类收益额外 `+1`
- 亲近类增长额外 `-1`

### 傲娇

- 普通召见的亲近增长 `-1`
- 送礼时亲近额外 `+2`

## 六、孩子出生后的早期养成

### 6.1 年龄推进

系统需要记录：

- `ageMonths`

每过一个月，孩子月龄 `+1`。  
显示年龄时，可按整岁展示：

- `ageYears = floor(ageMonths / 12)`

### 6.1.1 出生后的固定银两消耗

孩子出生后即进入固定养育开销：

- `0 ~ 35` 月：每月 `20`
- `36` 月及以上：每月 `35`

此外，孩子在以下节点必须结算礼仪支出：

- 满月礼：`180`
- 周岁礼：`260`
- 两岁礼：`320`

若玩家不支付该笔开销，则：

- 玩家声望 `-15`
- 该子嗣皇帝宠爱 `-3`

这样可以让皇嗣系统在三年测试周期内就形成稳定银两消耗。

### 6.2 三岁前的月度互动

在孩子三岁之前，每月固定触发一次“母子相处事件”，玩家可从三种教育方式中选择一种。

#### 温柔陪伴

- 亲近 `+5`
- 健康 `+2`

#### 启蒙开智

- 天赋 `+3`
- 亲近 `+2`

#### 严教立规

- 天赋 `+4`
- 亲近 `-3`
- 健康 `-1`

这些选项全部为硬规则，AI 只负责把月度相处写成对话框。

若玩家在三岁前选择更高规格的富养选项，则可额外付费：

- 添置衣物玩具：`60`
- 厚赏乳母嬷嬷：`120`
- 添置稀有启蒙物件：`260`

高规格投入主要提高：

- 亲近
- 后续服从倾向
- 少量皇帝对子嗣宠爱

## 七、皇嗣管理中的主动操作

玩家可在皇嗣管理中对自己所生的孩子进行主动操作。V1 推荐保留两项即可：

### 7.1 召见

- 每旬可召见一次
- 消耗 `1` 时间格
- 亲近 `+3`

### 7.2 召见并送礼

玩家在召见时可追加送礼，提高亲近。

- 小礼：`20` 两，亲近 `+3`
- 厚礼：`50` 两，亲近 `+6`
- 贵礼：`100` 两，亲近 `+10`

这样，皇嗣系统就能立刻具备可操作性，不必等全部朝堂系统都做完。

### 7.3 与偷情线子嗣的后续交互

若某子嗣 `isSecretLineage = true`，则：

- 皇嗣管理中的公开信息仍按“皇帝之子”显示
- 只有在后续特殊剧情中，系统才读取 `biologicalFatherId`
- 玩家可选择是否推动“父子相认”剧情

这类子嗣仍可被系统正常纳入皇嗣排行、召见、教育与立储流程；只有剧情判定会额外读取其真实血脉字段。

## 八、三岁后的教育系统

当孩子满三岁后，进入正式教育阶段。

### 8.1 重华宫教育

孩子可进入重华宫，按季度结算成长。

每季度效果：

- 天赋 `+4`
- 健康 `+2`
- 亲近 `+1`

### 8.2 卢安平辅导

你已经明确提出：孩子三岁后可选择让卢安平辅导，且卢安平每季度向玩家汇报一次成长情况。

推荐规则：

- 若选择卢安平辅导，则每季度：
  - 天赋 `+6`
  - 朝臣支持 `+2`
  - 亲近 `+1`

同时，每季度固定触发一次“卢安平汇报孩子成长”的对话事件。

### 8.3 为什么卢安平不直接加皇帝宠爱

卢安平更合理的作用是提升孩子的天赋和未来的朝堂竞争力，而不是直接让皇帝更偏爱。  
皇帝对子嗣的宠爱，仍应通过“孩子本身条件 + 生母影响 + 朝臣支持 + 玩家美言”来体现。

## 九、朝臣支持与后期竞争

你已经明确提出：后期宠爱也取决于朝堂中多少人支持该孩子。  
为了快速跑通，V1 先不做细颗粒派系，只保留一个总指标：

- `courtSupport`

此外，再补充两个用于游说和储君竞争的辅助指标：

- `familyFavor`：家族与玩家一方的支持度，范围 `0~100`
- `ministerFavor`：朝臣整体对该孩子的支持意愿，范围 `0~100`

其中：

- `familyFavor` 更容易被家族事务和玩家直接干预
- `ministerFavor` 更容易被政治、美言、银两接济和卢安平长期培养影响
- `courtSupport` 作为最终用于储君判定的总值，按季度刷新

### 9.0 统一说明

为了快速实现，V1 不做细分“哪位大臣支持哪位孩子”的名单系统，而是将朝堂支持压缩为总值。  
这会牺牲部分细节，但能让“立储竞争”先跑通。

## 十、储君候选与立储规则

### 10.1 候选资格

你已明确提出：

- 所有男性孩子都可以进入储君候选
- 高真心值妃子的女儿也可以进入

但这里必须服从更高优先级的血脉限制。  
原始总架构已经明确规定：

- 和亲公主与异国贡女所出的孩子，不可被立为储君
- 唯一例外是 `尘缘夙错` 线路

因此，V1 正式写为：

#### 男性子嗣

- 若生母不是和亲公主 / 异国贡女：
  - 只要成功出生并存活，即可进入储君候选池
- 若生母是和亲公主 / 异国贡女：
  - 默认不得进入储君候选池
  - 仅当当前线路为 `尘缘夙错` 时，才允许进入储君候选池

#### 女性子嗣

- 仅当其生母的 `皇帝真心值 >= 80` 时，才可进入储君候选池
- 但若其生母为和亲公主 / 异国贡女，则同样默认不得进入
- 仅当当前线路为 `尘缘夙错` 时，才允许进入储君候选池

这样既满足你的设定，也和原始总架构保持一致。

### 10.2 立储硬门槛

进入候选池，不等于可以立储。  
立储事件的触发，必须同时满足：

- 子嗣天赋值 `> 70`
- 子嗣宠爱值 `> 80`
- 子嗣朝堂支持 `> 80`

此外，子嗣必须：

- 存活
- 非死胎
- 未被系统移出皇嗣管理

### 10.3 为什么不额外加年龄硬门

为了快速跑通，这一版不额外增加“必须达到几岁才能立储”的硬门槛。  
古代皇嗣可以幼年立储，因此在系统层面直接允许即可，后续若需要再加剧情限制。

### 10.4 朝堂支持总值的季度刷新

为了让 `courtSupport` 和 `familyFavor / ministerFavor` 接得上，季度刷新时采用：

`courtSupport = clamp(0, 100, floor(familyFavor * 0.4) + floor(ministerFavor * 0.6))`

也就是说：

- 家族支持有用，但不是全部
- 真正决定能不能立储的，仍然是更广义的朝臣支持

## 十一、玩家帮助孩子争储的简化规则

你提出玩家可通过：

- 自己的政治属性
- 家族事务谋划
- 与朝堂臣子谋划
- 提供银两接济
- 政治加心计作为判定游说

来增加家族和其余臣子的好感度。

为了快速实现，V1 推荐只保留两种主动操作和一套统一公式。

### 11.1 家族事务谋划

玩家可通过“家族事务”扶持自己的孩子。

每次行动：

- 消耗 `1` 时间格
- 默认消耗 `30` 两
- 使用玩家的政治和心计做一次家族游说判定

公式：

`家族谋划值 = 30 + 玩家政治 / 2 + 玩家心计 / 40`

结果：

- 若成功：
  - `familyFavor +6`
  - 若玩家家世为高门（四品及以上、镇国公、和亲公主等），额外 `+2`
- 若普通成功：
  - `familyFavor +4`
- 若失败：
  - `familyFavor +1`

为了便于测试，V1 可先不做家族事务失败惩罚。

### 11.2 朝堂臣子谋划

玩家可在朝堂事务中设法争取臣子支持。

每次行动：

- 消耗 `1` 时间格
- 默认消耗 `50` 两
- 使用玩家的政治和心计做一次臣子游说判定

公式：

`臣子谋划值 = 25 + 玩家政治 / 2 + 玩家心计 / 50 + floor(提供银两 / 20)`

结果：

- 若谋划值 `< 50`
  - `ministerFavor +2`
- 若谋划值 `50~69`
  - `ministerFavor +4`
- 若谋划值 `70~89`
  - `ministerFavor +6`
- 若谋划值 `>= 90`
  - `ministerFavor +8`

### 11.3 皇帝面前为孩子美言

玩家也可以在与皇帝的交互中为孩子美言。

每次成功美言：

- 子嗣宠爱 `+3`

若玩家当前宠爱 `>= 60`，则：

- 额外再 `+2`

因此：

- 普通成功美言：`+3`
- 高宠状态下成功美言：`+5`

### 11.4 统一好处

这套写法的优点是：

- 玩家只要有政治和心计，就能推动争储
- 银两有直接用途
- 家世越高，家族事务越有优势
- 朝臣支持和皇帝宠爱两条线分开增长，系统更清晰

## 十二、教育系统的正式修正

你原话中提到“教育会影响孩子成年后的年龄”。  
这在逻辑上不成立，因此本版本正式修正为：

- 教育影响孩子成年后的能力、竞争力与服从倾向
- 教育不改变年龄推进

年龄仍然只随月份自然增长。

## 十三、三岁后的教育系统补充

### 13.1 重华宫教育

孩子满三岁后，进入正式教育阶段。  
玩家可选择把孩子送入重华宫接受教育，或由玩家亲自前往重华宫教学。

### 13.2 玩家亲自在重华宫教育

玩家可消耗自身资源来提高孩子能力。

每次亲自教育：

- 消耗 `1` 时间格
- 消耗 `2` 体力

收益由玩家主属性决定：

- 若玩家政治 `>= 60`
  - 孩子天赋 `+3`
- 若玩家政治 `< 60`
  - 孩子天赋 `+2`

同时：

- 孩子亲近 `+2`

若玩家本身亲近值已高于 `70`，再额外：

- 孩子亲近 `+1`

### 13.3 卢安平辅导的容量限制

你已经明确提出：卢安平每季度最多只能辅导 `3` 个孩子，而且辅导孩子越多，每个孩子加的属性越少。

正式规则如下：

#### 仅辅导 1 个孩子时

- 天赋 `+8`
- 朝臣支持 `+3`
- 亲近 `+1`

#### 同时辅导 2 个孩子时

- 每个孩子：
  - 天赋 `+6`
  - 朝臣支持 `+2`
  - 亲近 `+1`

#### 同时辅导 3 个孩子时

- 每个孩子：
  - 天赋 `+4`
  - 朝臣支持 `+1`
  - 亲近 `0`

#### 超过 3 个孩子

- 不允许进入卢安平辅导名单

### 13.4 卢安平季度汇报

每季度结束时，若孩子处于卢安平辅导中，则固定触发一次成长汇报事件。  
该事件只用于展示文本，不改动真实数值，真实数值由系统在季度结算时先完成。

## 十四、性格的正式作用

你已经明确提出：性格只影响孩子是否容易听从安排、是否容易叛逆，属于剧情变化，不直接大规模改数值。

因此 V1 正式写为：

- 性格主要影响“服从倾向”
- 不直接决定皇帝宠爱
- 不直接决定储君资格
- 只对部分养成操作做轻微修正

### 14.1 服从倾向

为便于后续剧情调用，给每种性格配一个服从倾向基础值：

- 温和：`80`
- 腼腆：`75`
- 活泼：`55`
- 圆滑：`65`
- 高冷：`45`
- 傲娇：`40`

服从倾向越高：

- 越容易听从玩家安排
- 越不容易在后续剧情中叛逆

服从倾向越低：

- 越容易出现违逆、顶撞、擅自行事等剧情

### 14.2 性格对日常培养的轻微修正

- 温和：亲近类成长 `+1`
- 腼腆：亲近类成长 `+1`
- 活泼：健康成长 `+1`
- 圆滑：朝臣支持成长 `+1`
- 高冷：天赋成长 `+1`，亲近成长 `-1`
- 傲娇：普通召见亲近 `-1`，送礼亲近 `+2`

## 十五、玩家可进行的朝臣支持操作

### 15.1 玩家拉拢朝臣

玩家可为自己的孩子拉拢朝臣支持。

推荐规则：

- 每次行动消耗 `1` 时间格
- 默认消耗 `50` 两
- 增加的朝臣支持取决于玩家政治值：
  - 政治 `< 30`：朝臣支持 `+2`
  - 政治 `30 ~ 59`：朝臣支持 `+4`
  - 政治 `60 ~ 79`：朝臣支持 `+6`
  - 政治 `80+`：朝臣支持 `+8`

### 15.2 在皇帝面前为孩子美言

玩家也可以在与皇帝的互动中为孩子美言。

为了便于实现，推荐规则：

- 每次成功美言：
  - 子嗣宠爱 `+3`
- 若玩家当前宠爱 `>= 60`：
  - 额外再 `+2`

即：

- 普通成功美言：`+3`
- 高宠状态下成功美言：`+5`

V1 中，这部分不需要复杂失败链，只要玩家满足与皇帝对话场景即可进行。

## 十六、子嗣宠爱在后期的刷新规则

孩子长大后，皇帝对子嗣的宠爱不再只看出生瞬间，而要开始吃到朝臣支持。

推荐在每季度结算时，刷新一次子嗣宠爱：

`子嗣当前宠爱 = clamp(0, 100, 10 + floor(子嗣天赋 * 0.4) + floor(子嗣容貌 / 100) + floor(max(0, 生母宠爱) / 5) + floor(朝臣支持 / 4) + 当季美言加成 + 皇帝心情修正)`

其中皇帝心情修正：

- `mood < 0`：`-5`
- `1 ~ 50`：`0`
- `51 ~ 100`：`+5`

其中：

- 当季美言加成为季度内累计值
- 季度结算后清空重新累计

这样会形成明确的后期趋势：

- 出生好 -> 起点高
- 培养好 -> 天赋与健康稳步上升
- 朝臣支持高 -> 后期更容易被皇帝重视

## 十七、整条系统的可实现闭环

到这里为止，整条线已经能够完整跑通：

1. 侍寝成功
2. 按福德判定怀孕
3. 第二天显性怀孕
4. 孕期每旬 `+10` 声望，并进入陪伴池
5. 若流产，则健康 `-60`，并按宫斗或自然流产处理后续
6. 若足月生产，则按健康值三档判定生产结果
7. 成功生育后创建皇嗣记录
8. 皇嗣进入皇嗣管理，可每旬召见并送礼
9. 三岁前按月发生母子相处事件
10. 三岁后进入重华宫或卢安平辅导
11. 玩家可通过拉拢朝臣与美言影响孩子未来竞争力
12. 每季度刷新孩子的成长与皇帝宠爱

这条链路中的所有真值都能由系统固定结算，不依赖 AI。

## 十八、桌面测试与结论

### 18.1 测试一：高宠爱高容貌的孩子

设定：

- 生母容貌 `920`
- 生母心计 `780`
- 生母宠爱 `70`
- 生母生产时健康 `680`
- 位分为妃
- 天赋随机值 `+12`

计算：

- 子嗣天赋 `= 20 + floor(780/20) + 12 = 20 + 39 + 12 = 71`
- 子嗣健康 `= 40 + floor(680/20) = 74`
- 子嗣容貌 `= 920`
- 初始朝臣支持 `= 15 + floor(70/15) = 19`
- 初始子嗣宠爱 `= 10 + floor(71*0.4) + floor(920/100) + floor(70/4)`
- 约为 `10 + 28 + 9 + 17 = 64`

结论：

这类孩子会有较高起点，但并没有直接爆表，后续仍需培养。

### 18.2 测试二：低位分但高心计母亲所生孩子

设定：

- 生母容貌 `650`
- 生母心计 `900`
- 生母宠爱 `25`
- 生母生产时健康 `360`
- 位分为美人
- 天赋随机值 `+15`
- 生产结果为难产

计算：

- 天赋 `= 20 + 45 + 15 = 80`
- 健康 `= 40 + 18 - 10 = 48`
- 容貌 `= 650`
- 初始朝臣支持 `= 5 + floor(25/15) = 6`
- 初始子嗣宠爱 `= 10 + 32 + 6 + 6 = 54`

结论：

虽然生母位分不高，但因为心计高，孩子天赋非常不错。  
这种设计能避免系统只看母亲位分，保留逆袭空间。

### 18.3 测试三：后期通过朝臣支持抬高皇帝宠爱

设定：

- 孩子三岁后，经过多次拉拢朝臣，朝臣支持达到 `60`
- 天赋 `75`
- 容貌 `880`
- 生母宠爱 `40`
- 当季美言加成 `+5`

季度刷新：

- 子嗣宠爱 `= 10 + floor(75*0.4) + floor(880/100) + floor(40/5) + floor(60/4) + 5`
- 约为 `10 + 30 + 8 + 8 + 15 + 5 = 76`

结论：

这说明后期确实可以通过玩家经营把孩子一步步抬成受重视的皇嗣，而不是出生时就完全定死。

### 18.4 测试四：流产后是否还能继续怀孕

设定：

- 角色流产一次
- 健康 `-60`
- 福德仍然 `> 0`
- 没有触发麝香导致的不孕后遗症

结论：

该角色可以继续参与后续怀孕判定。  
这和你要求的“若福德 > 0 则不影响后续生育”完全一致。

### 18.5 测试五：立储线是否可被主动经营

设定：

- 孩子天赋 `78`
- 当前子嗣宠爱 `74`
- familyFavor `60`
- ministerFavor `72`

季度刷新：

- `courtSupport = floor(60*0.4) + floor(72*0.6) = 24 + 43 = 67`

此时：

- 尚未达到立储门槛

若玩家连续两个季度执行：

- 一次家族事务谋划，平均 `familyFavor +6`
- 一次朝堂臣子谋划，平均 `ministerFavor +6`
- 两次皇帝前美言，子嗣宠爱 `+6~10`

则两季度后大致变为：

- familyFavor `72`
- ministerFavor `84`
- `courtSupport = 28 + 50 = 78`
- 子嗣宠爱 `80+`

再经过一季培养，就有机会达到：

- 天赋 `> 70`
- 宠爱 `> 80`
- 朝堂支持 `> 80`

结论：

这说明“立储”不是出生就锁死，也不是纯随机事件，而是可以通过玩家中期经营一步步推上去，适合测试。

### 18.6 测试六：卢安平辅导容量是否合理

若只辅导一个孩子：

- 一个季度天赋 `+8`

若辅导三个孩子：

- 每个孩子仅天赋 `+4`

结论：

这个落差足以让玩家做取舍：  
是集中资源冲一个继承人，还是平均培养多个孩子。

## 十九、适合写入总架构目录的统一语义

为了方便后续 AI 与代码实现理解，建议在总架构目录中统一描述为：

- 子嗣出生后拥有固定硬属性：天赋、宠爱、健康、容貌、亲近、性格、朝臣支持
- 子嗣容貌直接继承母亲生产时的容貌
- 子嗣天赋由母亲心计与随机数共同决定
- 皇帝对子嗣的宠爱由天赋、容貌、生母宠爱和后期朝臣支持共同决定
- 所有男性子嗣默认可进入储君候选，但若生母为和亲公主 / 异国贡女则默认排除，仅 `尘缘夙错` 线路例外；高真心值妃子的女儿也遵循同一条血脉限制
- 但若生母为和亲公主 / 异国贡女，则孩子默认不得立为储君，仅 `尘缘夙错` 线路例外
- 立储需要同时满足：天赋值 `> 70`、宠爱值 `> 80`、朝堂支持 `> 80`
- 玩家可通过家族事务、朝堂谋划、银两接济与在皇帝面前美言来推动孩子争储
- 三岁前按月触发母子相处事件
- 三岁后进入重华宫教育或卢安平辅导
- 卢安平每季度最多辅导三个孩子，辅导对象越多，每个孩子得到的收益越少
- 玩家也可亲自前往重华宫教育孩子，提高能力与亲近
- 玩家可在皇嗣管理中召见孩子、送礼、拉拢朝臣、在皇帝面前为孩子美言
- 若流产但福德仍大于 0，则不影响后续继续生育
- 若因麝香等原因进入不孕状态，则通过 `福德 < 0` 表现
