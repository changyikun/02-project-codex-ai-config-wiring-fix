# 核心硬规则与可玩循环 V1

## 0. 2026-04-24 覆盖口径
- 个人事务中的案件干预统一改为 `20 银两 -> 5 点嫌疑变化`。
- 若案件已锁定陷害妃嫔，玩家可持续执行 `20 银两 -> 该妃嫔嫌疑 +5`，达到 `100` 即定罪。
- 旁观案件同样统一为 `20 银两 -> ±5 嫌疑`，允许重复操作直到结案或银两不足。
- 月度用度采用三档策略：`节衣缩食 / 量入为出 / 锦衣玉食`，由玩家开局选择，后续在寝殿“调整用度”改动并于次月生效。
- 父亲官职改为阶梯月补：`0/2/4/6/8/10` 声望；入宫前 `6` 个月不会降位。
- 冷宫正式分为两条路径：事件打到负声望、自然连续两月低于 `-50`。
- 协理六宫 / 皇贵妃 / 皇后每年需额外支出 `260` 处理节庆。
- 后期夺位事件加入 `2000 / 4000 / 6000` 三档大额支出。

## 1. 文档目标
- 先把玩家真正能玩的部分做成稳定规则，再让 AI 负责表现。
- V1 只定四类硬系统：
  - 晋升
  - 怀孕
  - 毒杀
  - 调查定罪
- 后续补充的经济、皇嗣、皇帝与宫务规则，统一以各自专项硬规则文档为准
- 这四类系统全部采用“固定数值判定 + 可见反馈 + 可被玩家操作影响”的设计。

## 2. 总原则
- 所有核心结果先由本地规则引擎算出。
- AI 不负责判定是否怀孕、是否晋升、是否定罪。
- AI 只负责：
  - 把已发生的结果写成剧情
  - 为当前局面补 3 个选项文案
  - 根据结果输出更细腻的态度与情绪

## 3. 玩家可玩的最小循环

### 3.1 单旬循环
1. 旬开始：
   - 体力回到 `10`
   - 结算上旬遗留事件
   - 刷新夜间侍寝保底值
2. 自由行动：
   - 每消耗 `1` 时间格执行 `1` 次操作
   - 操作会改动体力、银两、数值、关系、案件状态
3. 夜晚阶段：
   - 判断是否侍寝、是否独寝、是否强制宫宴
4. 深夜阶段：
   - 若无侍寝则可继续行动
   - 执行案件推进、怀孕推进、临时状态清理
5. 旬结束：
   - 清空本旬行动 Buff
   - 立即切到下一旬清晨
   - 体力不继承上一旬剩余值，按“新旬基础体力 + 状态修正”重新计算；当前无修正时回到 `10`

### 3.2 月循环
- 每月下旬结束后执行：
  - 皇帝心情恢复/衰减
  - 怀孕推进
  - 压力扣寿
  - 伤病与毒伤推进
  - 晋升审查
  - 冷宫/接出判定
  - 月俸与月度用度
  - 子嗣养育与礼仪开销
  - 家族谋划与朝臣打点
  - 月结算文案

### 3.3 V1 可玩地点
- 寝殿：休息、读书、刺绣、调香、整理物品
- 御书房：研磨、诉苦、美言、撒娇
- 御膳房：购买食物、结识布自游
- 太医院：学药理、诊治、假死线前置
- 宝华殿：礼佛加福德
- 御花园：偶遇、口角、拉关系
- 后宫各宫：拜访妃子、送礼、试探、下手
- 地图空地点：进入地点后可展示本旬 NPC 交互入口。NPC 公共外出地点以地图热点配置为准，不只限于功能小场景。

### 3.5 NPC 旬级行动硬规则
- 每旬为 live 且非冷宫妃嫔生成一次主行动。
- 主行动可为留宫、公共外出、拜访玩家、拜访其他妃嫔、社交筹谋、敌意筹谋。
- 公共外出地点来自地图热点；“后宫”是聚合入口，玩家寝殿占位不作为 NPC 公共外出点。
- NPC 拜访其他妃嫔时，被拜访者本旬留殿接待，原本外出活动取消。
- NPC 拜访玩家必须带有目的，并在寝殿空闲时自动打断流程；NPC 先发起小对话，玩家必须选择接待、试探或拒绝。
- 公共地点遇见 NPC 时不得自动进入妃嫔对话；应先在地点上显示可点击的妃嫔交互入口，玩家主动点击后再用地点入场叙述和行动摘要进入妃嫔日常对话。
- 地图地点弹窗只展示地点自身描述，不得展示本旬 NPC 名字、摘要或动向，也不得新增“前去见某妃”直达选项；交互必须发生在进入地点后的场景中。公共外出的 `resolved` 只表示本旬已交谈，不得因此把 NPC 从目的地中移除。
- “后宫”是地图聚合地点，只能通过热点默认的 `进入此处` 进入后宫外景，不得额外提供“后宫总览”快捷选项。
- 后宫外景中的左侧工具面板关闭后必须回到后宫外景；工具面板内的 `外出` 必须直接回地图主视角，不得经过玩家寝宫。
- 外景回地图的状态清理必须晚于退出动画：先切地图视图，旧外景位置保留给旧场景淡出；退出完成后再清 `activeMapLocation` 和旧面板状态。
- 被其他 NPC 拜访且尚未收束的妃嫔必须在自己寝宫标记为“会客中”。
- 若行动表存在未收束的 `visit-consort`，目标妃嫔的位置真值优先级高于她自己的旧公共外出条目；她本旬必须显示为在寝宫会客，并不得同时出现在公共地点可交互名单中。玩家结束殿内会客后，来访者回自己的寝宫，目标不再“会客中”。
- 后宫住址匹配必须兼容 `旧居X` 这类历史 / 特殊住址，按 `X` 主殿处理。
- NPC 宫斗每旬全局只掷一次，默认 `10%`，每旬最多生成一条 NPC 宫斗案。

## 4. 时间与体力硬规则

### 4.1 体力
- 上限 `15`
- 每旬开始默认 `10`
- 下限 `0`
- 跨旬时必须重算体力，不得直接继承上一旬剩余值
- 当前默认公式：进入下一旬清晨后，体力回到 `10`；后续若接入怀孕 / 伤病 / 特殊状态，再在此基础上修正
- 寝殿内基础养成默认走低体力消耗口径：`诵读经典 / 泼墨作画 / 镂月裁云 / 调制香薰 = -1`，`习舞奏乐 = -2`，`请平安脉 = 0`，`殿内小酣 = +3`
- 体力为 `0` 时不可执行除 `小酣 / 进食 / 就寝 / 紧急应答` 外的行为

### 4.2 时间锁
- 每个地点有开放时间，关闭时不可进入
- 被强制事件占用时，本格不能自由行动
- 侍寝占用 `夜晚`，且直接跳过玩家的深夜自由行动
- 侍寝中的“美言 / 诉苦 / 抹黑”只属于夜晚对话，不计入主动宫斗，不扣福德，不进入调查
- 玩家在侍寝时可对交好妃子美言或对交恶妃子抹黑，目标宠爱 `±3~5`
- 若其他交好妃子侍寝，则有 `80%` 概率为玩家美言；若交恶妃子侍寝，则有 `20%` 概率抹黑玩家，玩家宠爱同样按 `±3~5` 结算

### 4.3 强制就寝
- 玩家主动选择 `就寝`，本旬立即结束
- 无论是主动就寝还是自然走到旬末，进入下一旬后都按新旬口径重算体力
- 若剩余体力 `<= 1` 且压力 `>= 85`，系统在深夜前给出强制休息警告

## 4.4 经济与银两硬规则

### 4.4.1 月俸

- 玩家每月按位分领取月俸
- `位分 < 妃` 且 `宠爱 1~19`：按 `70%` 发放
- `位分 < 妃` 且 `宠爱 <= 0`：按 `50%` 发放
- 冷宫中：月俸为 `0`
- `宠爱 >= 61`：当月实际月俸 `+20%`

### 4.4.2 倾情赠银

- 每月最多 `2` 名 `倾情 > 60` 的妃子赠银
- 每名赠送自己当月实际月俸的 `1/4`
- 赠银总额允许高于玩家基础月俸

### 4.4.3 固定银两消耗池

系统必须持续消耗玩家银两，至少包括：

- 月度用度开销
- 家族谋划
- 朝臣打点
- 子嗣养育与教育
- 稀有礼物与补品
- 宫斗案件运作

### 4.4.4 月度用度开销

设玩家当月实际月俸为 `S`：

- `节衣缩食`：用度 `floor(S * 0.25)`，声望 `-5`，健康 `-1`
- `量入为出`：用度 `floor(S * 0.5)`，声望 `0`，健康 `0`
- `锦衣玉食`：用度 `floor(S * 0.75)`，声望 `+10`，健康 `+1`
- 开局三选项只决定本月用度策略，不再给予一次性声望、宠爱或压力变化。
- 寝殿“调整用度”登记的是下月策略，本月结算仍按当前策略执行。

### 4.4.5 政治相关的大额运作

- `政治 < 20`：不可正式朝臣打点
- `政治 >= 20`：开放 `180 / 320 / 500` 三档朝臣打点
- 家族谋划当前按季度结算，仅保留第一档 `120`

### 4.4.6 案件干预

- 玩家个人事务：`20` 银两使嫌疑 `±5`
- 若已锁定陷害妃嫔：可对该妃嫔执行 `20` 银两使嫌疑 `+5`
- 旁观案件同样统一为 `20` 银两使嫌疑 `±5`
- 以上操作均可重复，直到结案或银两不足

## 5. 晋升、降位与冷宫规则

### 5.1 基础原则
- 声望决定 `应有位分`
- 月结算负责把 `实际位分` 向 `应有位分` 推进
- 自然晋升每月最多 `2` 级
- 自然降位每月最多 `2` 级
- “未侍寝不能晋升”正式废止

### 5.2 位分门槛
- 直接沿用设计文档中的 `18` 级位分与声望门槛
- 引擎中写成固定配置表，不允许交给 AI 推断

### 5.3 晋升资格硬门
- `prestige >= targetRank.requiredPrestige`
- `alive = true`
- `inColdPalace = false`
- `insane = false`
- 目标位分有空位
- 皇帝心情限制：
  - `mood < 0`：本月不晋升任何人
  - `mood < 20`：本月不可晋升到 `妃位及以上`
  - `mood < 50`：本月不可新立 `皇后`

### 5.4 自然推进规则
- 若应有位分高于实际位分：
  - 本月最多上升 `2` 级
  - 但仍需逐级检查空位、心情、特殊位开放条件与血脉封顶
- 若应有位分低于实际位分：
  - 本月最多下降 `2` 级
  - 降位不受低位空位限制

### 5.5 同位竞争顺序
- 当某高位分空位不足、多人同时满足进入条件时，按以下顺序排序：
  - 声望更高
  - 宠爱更高
  - 皇帝真爱值更高
  - 当前位分更高
  - 入宫时间更早

### 5.6 路线与血脉限制
- 和亲公主、异国贡女默认封顶 `皇贵妃`
- 仅 `尘缘夙错` 路线例外，允许进入皇后线
- 生母为和亲公主、异国贡女的孩子默认不得被立为储君，仅 `尘缘夙错` 线路例外

### 5.7 皇后与皇贵妃
- 皇后唯一，人数上限 `1`
- 皇贵妃为特殊位，人数上限 `1`
- 仅在以下任一条件成立时，才允许新设皇贵妃：
  - 当前无皇后
  - 皇后 `health < 400` 且连续 `3` 月
  - 皇后 `favor < 20` 且连续 `6` 月

### 5.8 冷宫
- 若因宫斗 / 惩处导致声望 `< 0`，则立即进入冷宫
- 若只是自然声望 `< 0`，则连续 `2` 月后进入冷宫
- 若声望重新回到 `0` 或以上，且无剧情强制冷宫标记，则月结算时自动接出

### 5.9 真爱值最高保护
- 若本次受罚角色为当前皇帝真爱值最高的妃子，则处罚后声望保底为 `100`
- 该保护只作用于处罚事件，不覆盖其他自然变动

### 5.10 玩家可见反馈
- UI 必须明确给出本月未晋升或降位原因：
  - 声望不足
  - 位分已满
  - 皇帝心情不足
  - 当前在冷宫
  - 血脉限制
  - 皇贵妃 / 皇后位未开放

## 6. 怀孕规则

## 6.1 触发前提
- 仅在玩家或 NPC 侍寝成功后触发
- 当前 `pregnancy.status = none`
- `fortune >= 0`
- `fertilityCooldownMonths = 0`
- 未处于永久不孕状态
- 未命中特殊剧情禁止受孕标记

## 6.2 怀孕概率
- 福德是主导值，其余项只做修正。

```txt
pregnancyChance =
  clamp(0, 95,
    max(0, fortune)
    + favorBonus
    + interestBonus
    + ageBonus
    + healthBonus
    - stressPenalty
    - infertilityPenalty
  )
```

### 6.2.1 修正项
- `favorBonus`
  - 宠爱 `1..20`：`0`
  - `21..40`：`5`
  - `41..60`：`10`
  - `61..80`：`15`
  - `81..100`：`20`
- `interestBonus`
  - 本次侍寝兴致 `< 40`：`0`
  - `40..69`：`5`
  - `70..89`：`10`
  - `90..100`：`15`
- `ageBonus`
  - `16..22`：`10`
  - `23..28`：`5`
  - 其他：`0`
- `healthBonus`
  - 健康 `>= 800`：`10`
  - `600..799`：`5`
  - 其他：`0`
- `stressPenalty`
  - 压力 `60..79`：`10`
  - `80..100`：`25`
- `infertilityPenalty`
  - 轻度受损：`20`
  - 重度受损：`50`
  - 永久不孕：`999`

## 6.3 怀孕状态推进
- 受孕成功后：
  - `status = suspected`
  - `monthCount = 1`
  - `fatherId = emperor`
- 次月月结算：
  - `status suspected -> confirmed`
- 第 `4` 月：
  - `status -> stable`
- 第 `8` 月：
  - `status -> late`
- 第 `10` 月月结算：
  - 触发生产事件

### 6.3.1 生育冷却
- 生产后：`fertilityCooldownMonths = 3`
- 流产后：`fertilityCooldownMonths = 3`
- 冷却期间即使发生侍寝，怀孕判定也直接失败

## 6.4 流产风险

```txt
miscarriageRisk =
  baseRiskByStage
  + stressRisk
  + poisonRisk
  - healthProtection
  - medicineProtection
```

### 6.4.1 分段
- `suspected / confirmed`：基础 `12`
- `stable`：基础 `6`
- `late`：基础 `10`

### 6.4.2 修正项
- `stressRisk`
  - 压力 `< 60`：`0`
  - `60..79`：`10`
  - `80..100`：`25`
- `poisonRisk`
  - 普通外伤/惊吓：`10`
  - 麝香命中：`100`
- `healthProtection`
  - 健康 `>= 800`：`10`
  - `600..799`：`5`
- `medicineProtection`
  - 有保胎照料：`5`
  - 高药理照料：`10`

## 6.5 福德硬门
- `fortune < 0`：
  - 本次侍寝受孕判定直接失败
  - 若已怀孕，则次月月结算直接流产

## 6.6 玩家可玩的怀孕干预
- 礼佛：提高 `fortune`
- 休息/减少宫斗：降低 `stress`
- 太医院安胎：降低 `miscarriageRisk`
- 下毒麝香：针对他人怀孕状态做破坏

## 7. 毒杀规则

## 7.1 毒物与成本
| 毒物 | 价格 | 基础效果 | 案件严重度 |
| --- | --- | --- | --- |
| 鹤顶红 | `500` 两 | 中毒成功则目标死亡 | `lethal` |
| 麝香 | `250` 两 | 怀孕者必定流产，未孕者增加不孕风险 | `high` |
| 陨颜丹 | `150` 两 | 容貌、气质永久下降 | `medium` |

## 7.2 发动前提
- 持有毒物
- 与目标存在接触路径：
  - 食物
  - 礼物
  - 同宫探访
  - 帮手代投
- 本格消耗 `1` 时间、`1` 体力

## 7.3 投毒成功率

```txt
poisonSuccessRate =
  clamp(5, 95,
    baseByPoison
    + mindGapBonus
    + medicineGapBonus
    + accessBonus
    + helperBonus
    + trustBonus
    - alertPenalty
  )
```

### 7.3.1 转换
- `mind(x) = floor(intrigue / 10)`，把 `0..1000` 转成 `0..100`

### 7.3.2 组件定义
- `baseByPoison`
  - 鹤顶红：`35`
  - 麝香：`50`
  - 陨颜丹：`60`
- `mindGapBonus`
  - `(attackerMind - targetMind) * 0.3`
- `medicineGapBonus`
  - `(attackerMedicine - targetMedicine) * 0.4`
- `accessBonus`
  - 食物/茶水：`15`
  - 赠礼：`10`
  - 同宫房内：`5`
- `helperBonus`
  - 无帮手：`0`
  - 普通宫女：`5`
  - 高忠诚帮手：`10`
  - 布自游倾情 `=100`：`20`
- `trustBonus`
  - 目标对你好感 `>= 40`：`10`
  - `>= 70`：`15`
- `alertPenalty`
  - 目标正在戒备：`10`
  - 目标近期遭遇过毒害：`15`
  - 目标受太医重点照看：`10`

## 7.4 毒物结果
- 鹤顶红：
  - 成功则目标在本旬内死亡
  - 直接开启 `lethal` 级案件
- 麝香：
  - 若目标怀孕，直接流产
  - 若未怀孕，增加 `infertilityPenalty = 20~50`
  - `10%` 概率触发“一尸两命”仅限怀孕后期
- 陨颜丹：
  - `appearance` 下降 `250 + successMargin * 2`，最大 `400`
  - `temperament` 同步下降 `150 + successMargin`，最大 `300`

## 7.5 失败后果
- 投毒失败不会无事发生：
  - 目标警觉度上升
  - 攻击者立即获得基础嫌疑
  - 若有目击者，直接建案

## 8. 调查与定罪规则

## 8.1 建案条件
- 出现以下任一情况时强制建案：
  - 中毒死亡
  - 妃嫔流产且存在人为痕迹
  - 公开容貌受损且受害人指控
  - 重要人物在短期内连续异常

## 8.2 案件结构
- 每个案件固定持续最多 `3` 旬
- 状态：
  - `collecting`
  - `investigating`
  - `verdict`
  - `closed`

## 8.3 初始嫌疑值

```txt
initialSuspicion =
  evidenceBase
  + purchaseTrace
  + motiveBonus
  + witnessBonus
  - concealmentScore
  - alibiScore
```

### 8.3.1 组件定义
- `evidenceBase`
  - 鹤顶红：`55`
  - 麝香：`40`
  - 陨颜丹：`30`
- `purchaseTrace`
  - 公开购买：`25`
  - 借他人之手：`10`
  - 黑市/神秘商贩：`5`
- `motiveBonus`
  - 宿敌/刚口角：`10`
  - 直接争宠对手：`20`
  - 已公开交恶：`30`
- `witnessBonus`
  - 无人：`0`
  - 模糊目击：`10`
  - 宫人证词：`20`
  - 关键证人：`35`
- `concealmentScore`
  - `floor(attackerIntrigue / 40) + floor(attackerMedicine / 10) + cleanupBonus`
- `alibiScore`
  - 弱不在场证明：`10`
  - 强不在场证明：`20`

## 8.4 每旬调查推进
- 案件开启后，每旬末所有嫌疑人自动增加一轮压力值：

```txt
investigationTick =
  severityPressure
  + imperialAttentionBonus
  + victimRankBonus
  - currentConcealment
```

### 8.4.1 组件定义
- `severityPressure`
  - `medium`：`5`
  - `high`：`10`
  - `lethal`：`15`
- `imperialAttentionBonus`
  - 皇帝心情低落且亲自过问：`10`
  - 普通关注：`5`
  - 未关注：`0`
- `victimRankBonus`
  - 嫔以下：`0`
  - 嫔/容华：`5`
  - 妃及以上：`10`

## 8.5 玩家在调查期可做的事
| 行为 | 时间 | 成本 | 固定效果 |
| --- | --- | --- | --- |
| 贿赂宫人 | 1 格 | `20~100` 两 | 自身嫌疑 `-10~-25` |
| 毁灭证物 | 1 格 | 失败风险 | 成功则嫌疑 `-15`，失败则 `+20` |
| 栽赃他人 | 1 格 | 需目标存在动机 | 成功则转移 `10~20` 嫌疑 |
| 求情 | 1 格 | 消耗关系 | 仅降低处罚，不清除定罪 |
| 找盟友作证 | 1 格 | 消耗好感/倾情 | 获得 `10~20` 不在场分 |
| 装病回避 | 1 格 | 低收益 | 仅拖延，不减少最终嫌疑 |

## 8.6 三旬结案阈值
- 第三旬末按最终嫌疑值结案：
  - `>= 100`：定罪
  - `80..99`：重嫌疑，皇帝视心情决定轻罚或搁置
  - `< 80`：疑案，案件关闭

## 8.7 定罪后处罚
| 案件等级 | 默认处罚 |
| --- | --- |
| `medium` | 禁足 `1` 月，声望 `-100~-200` |
| `high` | 降位 `1~2` 级，禁足，宠爱 `-10~-30` |
| `lethal` | 打入冷宫、赐死、剧情特赦三选一，由位分和剧情标记决定 |

## 8.8 皇帝心情影响
- 凶案一旦定罪或确认死亡：
  - 当月后宫每死 `1` 人，皇帝心情立即下降
- 半年内累计死亡 `>= 6`：
  - 皇帝心情直接 `-50`

## 9. 四套规则的先后顺序
- 同一轮中若同时触发多个系统，按以下顺序结算：
1. 强制事件
2. 投毒/伤害结果
3. 建案与初始嫌疑
4. 夜间侍寝
5. 怀孕判定
6. 旬末案件推进
7. 月末晋升、流产、寿命结算

## 10. V1 最重要的开发约束
- 晋升、怀孕、毒杀、定罪必须可重放：
  - 相同输入得到相同结果
  - 所有中间值可追踪
  - AI 不可覆盖判定
- 前端要给玩家“为什么会这样”的解释：
  - 成功率来源
  - 门槛未满足原因
  - 案件嫌疑变化原因
