# 宫斗事务硬规则（架构目录版）

## 一、文档定位

本文件定义玩家与 NPC 共用的宫斗事务规则。  
系统负责真实数值，AI 只负责：

- 娇娇的通报
- 养心殿对话
- 与被害妃嫔、陷害妃嫔的后续剧情

## 二、宫斗行为类型

### 2.1 造谣

- 欺凌宫人：轻
- 奢侈浪费：轻
- 与人偷情：中
- 不敬先祖：重
- 意图谋逆：重

### 2.2 下毒

- 陨颜丹：轻
- 麝香：中
- 鹤顶红：重

### 2.3 玩家下毒投放 QTE

玩家主动选择“下毒”时，先进入投放表现阶段。
该阶段只判断“是否把毒投放到杯盏时机中”，不替代后续行动检定和隐匿检定。

- 玩家必须先持有本次选择的毒药；当前可用毒药为陨颜丹、麝香、鹤顶红。
- 毒药来源收束到掖庭院“月姑姑”交易入口，购买后进入背包。
- 药理越高，QTE 成功区域越宽
- 心计越高，QTE 指针速度越慢
- 压力越高，QTE 指针速度越快
- 初始属性下应保持偏难，避免玩家在零药理 / 中低心计时稳定命中
- QTE 成功：消耗对应毒药 `1` 份，登记 `pending_resolution` 宫斗案，等待当旬结算
- QTE 失败：本次投放未成，不登记案件，也不消耗毒药
- 同一次 QTE 成功后只能登记一次案件，重复点击完成不得绕过消耗或重复建案

## 三、时间流程

- 白天选择宫斗行为
- 当旬夜晚统一结算
- 下一旬由娇娇通报结果
- 若暴露，则进入最多 `3` 旬调查
- 玩家相关重案会被带往养心殿

## 四、两次检定

每次宫斗固定两次检定：

1. 行动检定：决定行为是否生效
2. 隐匿检定：决定是否暴露并进入调查

四种标准结果：

- 生效且未暴露
- 生效但暴露
- 未生效但未暴露
- 未生效且暴露

## 五、基础公式

### 5.1 攻击值

- 造谣：`A = 发动者心计 / 10`
- 下毒：`A = 发动者药理`

### 5.2 防御值

- 造谣：`D = 目标心计 / 10 + 目标宠爱 / 20`
- 下毒：`D = 目标心计 / 10 + 目标宠爱 / 20 + 目标药理 / 4`

### 5.3 行动成功率

```txt
行动成功率 = 50 + A - D - 事件修正 - 嫁祸修正
```

- 轻：`0`
- 中：`5`
- 重：`10`
- 嫁祸：额外 `12`
- 最终截断：`10~90`

### 5.4 隐匿成功率

```txt
隐匿成功率 =
50
+ 发动者心计 / 12
- 目标心计 / 15
+ 发动者宠爱 / 8
+ 合谋修正
- 事件修正
- 嫁祸修正
```

- 轻：`5`
- 中：`10`
- 重：`15`
- 合谋：`+5`
- 嫁祸：额外 `15`
- 最终截断：`5~85`

## 六、合谋、嫁祸与福德

### 6.1 合谋

- 条件：好感 `> 60`
- 合谋后攻击值：

```txt
floor((发动者攻击值 + 合谋者对应攻击值) / 2) + 10
```

- 隐匿 `+5`
- 案件成立时初始获罪率 `+10`

### 6.2 嫁祸

- 行动检定 `-12`
- 隐匿检定 `-15`
- 初始获罪率 `+15`
- 调查期间每旬额外 `+5`

### 6.3 主动宫斗对福德的影响

- 主动造谣：福德 `-5`
- 主动下毒：福德 `-10`

## 七、调查、嫌疑人与定案率

### 7.1 进入调查

隐匿检定失败即进入调查。

### 7.2 嫌疑人结构

进入调查的案件必须保存 `suspects` 数组，最多 `3` 人。每名嫌疑人包含：

- `id`
- `subjectType: 'player' | 'consort'`
- `subjectId`
- `name`
- `suspicionRate`
- `isActualActor?`
- `isFramed?`
- `reason`

`convictionRate` 不再是唯一裁判，只作为当前案件最高 `suspicionRate` 的派生展示值。

### 7.3 初始定案率

```txt
实际发起者初始定案率 =
基础值
+ (100 - 隐匿成功率) / 2
+ max(0, 目标宠爱 - 20) / 2
+ 嫁祸值
+ 合谋值
- 发动者宠爱 / 5
```

基础值：

- 轻：`20`
- 中：`40`
- 重：`65`

嫌疑名单生成顺序：

- 强制优先纳入实际发起者
- 强制优先纳入被嫁祸者；被嫁祸者初始定案率至少 `70`
- 如果玩家是被害者或被嫁祸者，玩家必须进入嫌疑名单
- 其余名额从存活且非冷宫妃嫔中按动机分补足
- 总人数不超过 `3`

### 7.4 每旬增长

- 轻：`+8`
- 中：`+14`
- 重：`+20`
- 被嫁祸者额外：`+5`

调查推进时，所有嫌疑人的 `suspicionRate` 同步增长，并重新计算案件展示用的最高 `convictionRate`。

### 7.5 三旬结案与待裁断

任一嫌疑人的 `suspicionRate >= 100` 时，案件不立即定罪，而是进入待裁断：

```txt
pendingVerdictSuspectId = 首个达到 100 的嫌疑人 id
status = pending_verdict
outcome = pending
resolutionSummary = 已达定罪门槛，待养心殿裁断
```

- 满 `3` 旬仍无人达到 `100`：案件归档为疑案
- 疑案结案时写入 `archivedXunKey` 与 `resolutionSummary`
- 玩家相关待裁断案在下一旬清晨优先进入养心殿裁断事件
- NPC-only 待裁断案不打断玩家，按内廷裁断静默结案并应用对应 NPC 惩罚

## 八、宫斗事务中的调查面板

调查入口位于“宫斗事务”内，不新增左侧常驻按钮。

- 没有调查中、待裁断或已归档案件时不显示“调查”步骤；仅待夜晚结算的 `pending_resolution` 案件不显示空调查页
- 有调查中案件时显示“调查案件”
- 有 `pending_verdict` 案件时显示“待裁断案件”
- 有已结案件时显示“归档案件”
- 调查中案件展示：案件摘要、受害者、方式、物品、已查旬数、三名嫌疑人、每人定案率和嫌疑原因
- 待裁断案件展示：案件摘要、定罪候选人和“待养心殿传唤裁断”状态；不能继续银两干预
- 归档案件展示：定罪者或疑案结论、最终嫌疑分布、处理摘要

### 8.1 压低嫌疑

- `20` 银两：任一嫌疑人定案率 `-5`
- 银两不足、案件已结或嫌疑人不存在时不能操作

### 8.2 推高嫌疑

- `20` 银两：任一嫌疑人定案率 `+5`
- 若该嫌疑人定案率达到 `100`，进入 `pending_verdict`，等待养心殿或内廷裁断

### 8.3 干预次数

调查面板中的银两干预不设固定次数上限。
只要案件未结案、玩家仍有银两，就可以继续操作。

## 九、旁观案件与玩家牵连案件

玩家可以在“宫斗事务”的调查页干预任一调查中案件的任一嫌疑人。玩家被害、被嫁祸或主动宫斗暴露时，案件仍在调查页展示；若定案率达到 `100`，下一旬清晨由内侍传旨进入养心殿裁断，而不是由娇娇或旬报直接宣布定罪。

## 十、NPC 互斗

NPC 妃嫔之间也会使用同一套宫斗公式，但触发入口不再按每个 NPC 每旬单独掷骰。

当前代码口径：

- 每旬全局只掷一次 NPC 宫斗概率，默认 `10%`。
- 未命中全局概率时，本旬不会因为 NPC 数量多而额外补掷。
- 命中后，从 `npcActivity` 中的 `hostile-plot` 或高 `tension + ambition + stress` 动机妃嫔中选一名发起者。
- 每旬最多生成 `1` 条 NPC 宫斗案。
- 目标优先来自敌意筹谋行动的目标、关系矩阵中紧张较高的对象，或与发起者关系恶化的玩家。
- NPC 攻击其他妃嫔时，可以把玩家列为嫁祸对象；若玩家成为被害目标或嫁祸对象，案件进入调查时必须标记为玩家相关案件。
- 行动一旦建案，后续仍进入本文件定义的行动检定、隐匿检定、调查、定罪与养心殿规则。

这一规则用于避免“妃嫔数量越多，每旬宫斗越爆炸”的问题。NPC 宫斗要保持宫廷暗流存在，但不能让玩家每旬都被新案淹没。

## 十一、养心殿

以下情况会把玩家带到养心殿：

- 玩家相关案件进入 `pending_verdict`
- 玩家主动宫斗后暴露并被推至定罪门槛
- 玩家被他人陷害或嫁祸，且被推至定罪门槛
- 玩家是被害妃嫔，且案件进入待裁断

触发时机：

- 下一旬清晨优先触发，先于普通清晨通报、旬报和其他待播事件
- 先回到玩家寝宫，由内侍传旨：“养心殿传旨，请娘娘即刻前往听裁。”
- 玩家确认前往后，`activeMapLocation` 才切到真实 `养心殿`
- 裁断使用专用御前裁断背景 `yangxin_verdict_daytime.png` 和全局对话舞台；普通进入养心殿仍使用地点外景和入场剧情，不与裁断背景共用
- 内侍、皇帝、娇娇、玩家或妃嫔按发言人显示立绘
- 裁断结束后仍停留在 `养心殿`，由玩家自行外出或回宫，不自动回寝殿
- 玩家本人涉案时对白称呼使用“你 / 娘娘”，不在对话正文显示玩家本名
- 不再通过地图或寝殿工具面板主动打开旧裁断入口

养心殿裁断只负责：

- 求情
- 对最终惩罚做有限修正
- 写入定罪与处罚结果

裁断选择按玩家身份分流，不再只做统一三档减罚。

若玩家本人是定罪候选人：

- 据证自辩：减罚最多，但受害者关系风险最高
- 陈明疑点：温和指出案中疑处，中等减罚
- 伏身求宽：承认处境，请求从轻，稳定减罚
- 攀指旁人：把疑点引向旁人，可能减罚，但留下关系隐患
- 沉默领罪：不主动减罚，避免额外言语冲突

若玩家不是定罪候选人，只是受害者、被嫁祸者或相关旁听：

- 请皇上严断：加重处罚，受害者一方更认可
- 只陈案情：只补事实，不主动加重或减轻
- 指出疑点：小幅减轻处罚，关系波动较低
- 代为求情：明确替定罪候选人求宽，减罚较多，但得罪受害者一方
- 沉默旁听：不介入裁断，避免额外关系冲突

求情只减罚，不翻案。裁断完成后才写入：

- `convictedSuspectId`
- `verdictSummary`
- `penaltyApplied`
- 玩家或 NPC 的声望 / 宠爱 / 压力惩罚
- 相关关系变化

## 十二、适合写入总架构目录的统一语义

- 宫斗固定分为造谣与下毒两类
- 每次宫斗固定进行“行动检定 + 隐匿检定”
- 主动造谣扣福德 `5`，主动下毒扣福德 `10`
- 调查期固定最多 `3` 旬，案件以最多三名嫌疑人的独立定案率推进
- 宫斗事务调查页中的案件干预统一为 `20 银两 -> 5 点定案率变化`
- 任一嫌疑人的定案率达到 `100` 进入待裁断；三旬无人到 `100` 则归档为疑案
- 玩家相关待裁断案由下一旬清晨养心殿对话完成定罪和处罚；NPC-only 案件可由内廷静默裁断
- NPC 之间每旬由全局低概率敌意筹谋生成宫斗候选，形成持续但低频的宫斗氛围
