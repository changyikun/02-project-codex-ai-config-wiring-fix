# Changelog

## 维护规则

- 后续 changelog 按版本更新，不再按日期新增章节。
- 新增内容只写入最新版本章节；除格式迁移外，不回改旧版本章节。
- 如果最新版本中后续修正了该版本里已经写过的条目，不覆盖原条目原文；在最新版本末尾追加修订记录，格式为：`版本号 - 修改内容简述 - 见第 N 条`。
- 每个版本内的主要变更使用编号条目，便于修订记录引用。

## v0.4.0 - NPC 行动与外景视图收口

> 覆盖原 `2026-06-03` 与 `2026-06-04` changelog 内容；未发现 `2026-06-05` changelog 条目。

1. 修复非寝宫场景左侧 UI 与寝宫状态耦合的问题：大地图常驻“嫔妃 / 查看 / 纪事 / 情缘”改为地图覆盖面板，不再先把玩家传回寝宫；外景面板中点击“外出”仍返回地图主视角。
2. 调整数值 toast 口径：NPC-NPC 交流导致的妃嫔压力变化仅静默写入状态与关系矩阵，不再作为玩家清晨 / 结算 toast 展示；妃嫔声望变化仍可进入 toast。
3. 将本地个人测试目录 `__test__/` 与开发截图输出 `output/` 加入 `.gitignore`，避免同步每个开发者的临时测试信息。
4. 新增妃嫔旬级行动表与 NPC-NPC 关系矩阵：每旬为存活且非冷宫妃嫔生成留宫、公共外出、拜访玩家、拜访妃嫔、社交筹谋或低频敌意筹谋。
5. NPC 公共外出地点改为从地图热点配置派生，不再只覆盖妙音堂、御膳房、太医院、宝华殿、华清池；御花园、正阳门、重华宫、御书房、宫门、建章宫、冷宫、养心殿等空地点也可出现本旬妃嫔动向。
6. 公共地点偶遇优先读取本旬 NPC 行动表；无安排时不再从全体 live 妃嫔里随机抓人，避免“所有地点都能硬刷同一批 NPC”的假随机。
7. NPC 拜访其他妃嫔会让被拜访者本旬留在殿内接待；玩家进入被拜访者宫殿时可看到两名妃嫔同场与互动摘要。
8. 调整 NPC 遇见方式：NPC 拜访玩家会在寝殿空闲时自动打断并给出选择；公共地点和空地图点的 NPC 改为可点击交互，点击后用地点入场叙述和行动摘要进入妃嫔日常对话。
9. 修复外出 NPC 目的地可见性：地图地点弹窗只显示本旬在该地的妃嫔动向，不新增“前去见某妃”直达选项；玩家进入地点后再看到可交互妃嫔。`resolved` 只表示本旬已交谈，不再把 NPC 从目的地移除。
10. 后宫宫殿中，若某妃嫔本旬在自己殿中接待来访者，会在殿位和拜访入口标记为“会客中”。
11. NPC 动向调试只输出到浏览器控制台：进入寝殿或地图时用 `[npc-activity]` 和 `console.table` 展示本旬妃嫔行动，不新增游戏内调试层。
12. 收口 NPC 位置真值：公共外出或拜访他人的妃嫔不再能在自己寝宫被拜访；NPC 拜访玩家每旬最多 1 人且概率降低，NPC 拜访其他妃嫔时同一目标每旬最多接待 1 人。
13. 修复后宫作为地图地点时的返回语义：从地图进入后宫后，侧栏“外出”返回地图主视角，独立“回宫”才直接回寝殿，不再先经过玩家住所。
14. 修复 NPC 动向 debug 与后宫场景不一致的问题：未收束的 `visit-consort` 目标妃嫔按“在自己寝宫会客”显示，并从公共地点可见名单中排除；旧居类住址会归一到对应宫殿主殿。
15. 修复玩家结束 NPC 会客后的位置收束：`visit-consort` 被玩家会客结束后标记为已触发，来访妃嫔回到自己的寝宫，被拜访者不再显示“会客中”。
16. 后宫拜访妃嫔接入统一过夜链路：深夜拜访结束后由娇娇提醒、黑屏过夜，再进入清晨通报；熬夜惩罚改为在清晨通报中由娇娇说明并扣除，压力 +2、健康 -10 显示值、气质 -10 显示值。
17. NPC 宫斗改为每旬全局一次低概率敌意筹谋入口，每旬最多生成一条 NPC 宫斗案；宫斗候选来自 `hostile-plot` 行动或高动机妃嫔。
18. 朝堂事务与宫斗事务文案拆清：宫斗事务会登记真实案件；朝堂事务当前只暂存草案，不改数值、不进结算。
19. SaveGameV1 可选扩展 `progress.npcActivity` 与 `relations.npcRelationMatrix`，旧存档缺失时按空状态初始化，不提升 schema version。
20. 修复合并冲突：合并 `ChamberMainView` 中“吩咐娇娇”寝殿面板与 NPC 地点偶遇 / 拜访渲染链路，保留外景 NPC 可点击交互、NPC 拜访玩家、地点子场景和寝殿对白规则。
21. 修复合并后的测试口径：竖排寝殿行动按钮按无障碍按钮名断言，不再按拆分文本节点查找。
22. 修复后宫外景与寝殿状态耦合：从后宫打开“嫔妃 / 查看 / 纪事 / 情缘”等左侧工具面板后，关闭会回到后宫总览，不再落到空白外景或玩家寝殿。
23. 修复其他场景点击左侧“外出”的中转问题：处于后宫等外景工具面板时，“外出”直接返回地图主视角，不再以玩家寝宫作为中转。
24. 删除地图后宫热点中多余的“后宫总览”快捷选项，后宫入口统一使用默认“进入此处”进入外景。
25. v0.4.0 - 外景回地图的清理时机延后到退出动画完成，避免视觉上短暂闪回玩家寝宫 - 见第 23 条。
26. 调整影落掖庭初始住处：玩家正式寝殿从掖庭院迁至储秀宫西偏殿；掖庭院改为普通地图地点，开局定下用度后由娇娇带玩家经过后宫，直接触发陈婉宁初见，再回新寝殿。
27. v0.4.0 - 开发期存档策略改为不做旧结构迁移；`SaveGameV1` 结构不兼容时直接删除旧 envelope，移除旧 raw store、旧住处和旧字段 fallback - 见第 19 条。
28. v0.4.0 - 影落掖庭开局不再跳过娇娇地图引导；定下用度后先展示整张地图说明，完成后再前往后宫触发陈婉宁初见并回新寝殿 - 见第 26 条。
29. 后宫宫殿总览中，玩家自己的殿位新增“回到当前寝殿”入口，可从后宫殿位布局直接回家。
30. 全局屏蔽普通 UI 文本、图片、SVG、canvas、video 的选中和拖拽提取；输入框与可编辑区域仍保留文本选择能力。
31. v0.4.0 - 大地图移除独立 `椒房殿` 热点，原错误点位改回 `掖庭院`；回寝殿统一通过侧栏 `回宫` 或后宫玩家殿位入口 - 见第 26 条。
32. v0.4.0 - 大地图地点弹窗不再展示本旬 NPC 名字、摘要或动向；NPC 只在进入地点后的场景中作为可交互对象出现 - 见第 9 条。

## 2026-06-02

- 修复剧情 / 对话文本显示时背景 UI 仍可被点击的问题；共享对话舞台新增透明交互锁层，寝殿与地图主入口补充运行时点击保护。
- 修复共享对话进入选项态后，正文仍可点击并显示鼠标焦点框的问题；选项态下正文只保留可读，不再参与推进。
- 调整寝殿夜晚推进顺序：普通行动先展示本次行动结果，再展示夜晚侍寝通报或侍寝事件。
- 拆分数值 toast：每个数值差异独立生成一条 toast，最大可见数量调整为 10。
- 调整数值 toast 触发时机：数值变化先入队，等对应事件对白、夜晚通报或旬月通报实际出现后再显示。
- 调整侍寝反馈：他人侍寝导致玩家宠爱变化时，只保留状态变更与通报，不再弹宠爱 toast。
- 移除体力数值 toast，体力变化只通过常驻状态栏体现。
- 增加清晨通报去重保护，避免过夜推进过程中同一旬清晨通报被重复生成或重复展示。
- 移除宠爱相关 toast，玩家宠爱与妃嫔宠爱变化不再弹出数值提示；声望相关变化继续进入 toast。
- 寝殿主界面的“调整用度”新增第四个说明选项，由娇娇解释三档用度后返回选择，不写入策略状态。
- 修复玩家与妃嫔声望扣减不能低于 0 的问题；声望统一按 `[-2000, 5000]` 限制，为负声望与冷宫规则保留真实入口。
- 修复新建路线 / 新存档时旧局时间、旬月通报、宫斗案件、侍寝记录、交易记录等持久状态未被覆盖的问题；路线选择现在从初始状态重建一局。
- 启动页“开始”接入二级确认；确认后清空旧存档、重建初始状态并创建新存档。
- 启动页“回溯”接入单槽读档，可读取上一次 `SaveGameV1` 并恢复到对应游戏阶段；无存档时显示提示。
- 梳理启动菜单“前尘 / 回溯”现状：`回溯` 已接入读档，`前尘` 仍为按钮预留文案，未接入独立玩法。
- 新增 `docs/save-system-maintenance.md`，统一记录存档 key、SaveGameV1 envelope、新局 / 回溯入口和字段维护要求。
- 更新相关测试、交接文档、系统拆解文档与 `codex-workdocs` 记录。
- 修复妙音堂听曲数值 toast 延迟到旬月通报才显示的问题：听曲先结算压力变化，再推进时间，并在妙音堂行动结果出现时触发本次行动 toast。
- 接入系统宫宴第一版闭环：宫宴报名开启会生成独立事件通报，妙音堂曲谱报名写入 `PalaceBanquetProgressState`，3 月上旬傍晚自动结算宫宴表现和声望变化。
- 新增宫宴季调度与结算 runtime，曲谱品质、乐理、听曲次数和连翘关系会影响宫宴完成度；宫宴结果写入存档并防止同届重复触发。
- 修复时间通报只绑定寝殿主界面的问题；地图页现在也会展示未读的娇娇时间通报 / 系统事件通报，并在通报显示时屏蔽地图点击。
- 修正睡觉收束规则：普通外出行动推进到深夜时保留最后一次深夜行动；深夜行动后或体力归零时，先由娇娇提醒回宫 / 睡觉，再复用寝殿黑屏过夜转场进入次日清晨通报链路。
- 删除独立行动值状态字段和后端旧 runtime schema 中的对应校验，常规行动预算统一只看体力。
- 修复全局对话锁与场景按钮混用时的不可点击问题：侍寝互动和妃嫔会面对白需要先收起，再开放场景内固定操作。
- 修复建章宫太后场景仍被全局对话锁遮挡固定交互的问题：建章宫空闲态不再常驻渲染 `GlobalDialogueStage`，只在送礼问安 / 告辞后进入对话态，并在对话未收束前禁用固定按钮。
- 道具获得 / 失去接入数值 toast 队列，新增物品也会按缺省 0 数量计算增减，不再只提示已有数值字段。
- 修复外景场景侧栏语义：地点内 `外出` 明确返回地图主视角，独立 `回宫` 入口直接回寝殿；在寝殿面板内点击“外出”仍会先关闭面板并展示出行提示，避免转场卡住。
- 新增地点行动统一流程：妙音堂、太医院、御膳房、宝华殿、华清池的耗时行动统一走“行动结果对白 / NPC 对话 -> 深夜或体力归零判定 -> 娇娇提醒 -> 黑屏过夜”链路，不再各自直接 `advanceTime(1)`。
- 修复妙音堂普通听曲后行动结果对白消失的问题；普通地点行动现在复用共享行动结果舞台，数值 toast 随该舞台实际出现触发。
- 更新地点 / 侍寝相关测试预期，并让资源一致性测试支持 URL 编码的 public asset 路径。
- 新增曲谱练习闭环：玩家持有曲谱后可在妙音堂找连翘学谱，学习进度受玩家乐理、曲谱难度和连翘关系影响；曲谱登记宫宴不再消耗库存，练谱提升表现上限，宫宴当天再随机生成本场表现分。
- 修正乐理属性落点：不新增玩家属性字段，继续沿用既有 `talent` 字段承载乐理数值；界面与规则文案显示为“乐理”，读档时仅将历史过渡 `music` 归并回 `talent`。
- 调整学谱成长：每次向连翘学谱会同步增加乐理显示值 `+2`，并进入数值 toast。
- 系统宫宴报名开启调整为宫宴前一个月，首届宫宴会在 1 年 2 月上旬清晨进入报名期，避免开局提醒早于玩家可操作时间。
